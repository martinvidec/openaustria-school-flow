# Why this audit exists
The prior audit (260501-scd) ran the full pattern set (A–G) but explicitly classified
`getOrThrow('KEYCLOAK_ADMIN_CLIENT_SECRET')` call sites as FALSE_POSITIVE — code that READS
from env, not assigns. That is correct, but it does NOT prove the secret VALUE was never
committed. A separate, value-aware scan is required: take the current live value from `.env`
into a shell variable (NEVER echo) and run `git log -S "$KCSEC"` over all refs.

# Live-value access discipline (CRITICAL)
The local `.env` value of `KEYCLOAK_ADMIN_CLIENT_SECRET` MUST NOT appear in:
- The chat transcript (no `echo "$KCSEC"`, no `cat .env` without `sed` redaction)
- Any tracked file (FINDINGS.md, RUNBOOK.md, this PLAN.md)
- Any new commit by this audit
- Any process listing (`ps auxw` shows argv — pass via env, not argv)

Allowed:
- Loading into an unexported shell variable: `KCSEC="$(grep '^KEYCLOAK_ADMIN_CLIENT_SECRET=' .env | cut -d= -f2-)"`
- Computing 4+4 redaction for the report: `KCSEC_REDACTED="${KCSEC:0:4}…${KCSEC: -4}"`
- Computing a SHA-256 fingerprint for cross-reference: `echo -n "$KCSEC" | sha256sum | cut -c1-16`
- Passing via `git log -S "$KCSEC"` — the variable is expanded by the shell into git's argv
  in a single short-lived process; not echoed back. Acceptable risk per the constraint
  ("shell variable expansion is not transcript-logged unless the executor `echo`s it").

If `.env` has no `KEYCLOAK_ADMIN_CLIENT_SECRET` line, OR the value is empty/placeholder
(`changeme`, `<...>`, `${...}`), the value-search step is skipped and FINDINGS.md notes
"no live secret to compare; only variable-name scan performed."

**Layer 1 — Variable-name scan (always run, value-agnostic).**

Use git's pickaxe + grep to enumerate every commit where any line referencing
`KEYCLOAK_ADMIN_CLIENT_SECRET` was added or removed:

```bash
# All refs, every commit, with patch context.
git log --all -p -G 'KEYCLOAK_ADMIN_CLIENT_SECRET' -- '*' > /tmp/9iu-varname-hits.diff

# Compact list of commits + dates + subjects for the report:
git log --all --oneline -G 'KEYCLOAK_ADMIN_CLIENT_SECRET' --format='%h %ad %s' --date=short \
  > /tmp/9iu-varname-commits.txt
```

For each commit found, classify the diff hunk:

- **PLACEHOLDER** — value is empty (`KEYCLOAK_ADMIN_CLIENT_SECRET=` with nothing after `=`),
  literal `changeme` / `your-secret-here` / `<value>` / `xxx` / `placeholder`, or
  Bash-style default `${KEYCLOAK_ADMIN_CLIENT_SECRET:-default-placeholder}`.
- **ENV-VAR-REFERENCE** — code/config that reads the var without assigning a literal:
  `process.env.KEYCLOAK_ADMIN_CLIENT_SECRET`,
  `configService.getOrThrow('KEYCLOAK_ADMIN_CLIENT_SECRET')`,
  `${{ secrets.KEYCLOAK_ADMIN_CLIENT_SECRET }}` (GitHub Actions),
  `${KEYCLOAK_ADMIN_CLIENT_SECRET}` (docker-compose interpolation),
  `KEYCLOAK_ADMIN_CLIENT_SECRET=$KEYCLOAK_ADMIN_CLIENT_SECRET` (shell pass-through).
- **TRUE_POSITIVE** — actual secret literal: `KEYCLOAK_ADMIN_CLIENT_SECRET=<high-entropy-string>`
  where the right-hand side is ≥16 chars of base64url/hex/UUID-shape and matches none of the
  PLACEHOLDER patterns. Heuristic: hex32, hex64, UUID-format, or base64url ≥22 chars.

Write the per-commit classification into FINDINGS.md as a table.

**Layer 2 — Live-value scan (only if .env has a non-empty, non-placeholder value).**

```bash
# Load the live value into a shell variable WITHOUT echoing it.
# (Read line, strip the `KEYCLOAK_ADMIN_CLIENT_SECRET=` prefix; do not print.)
if [ -f .env ] && grep -q '^KEYCLOAK_ADMIN_CLIENT_SECRET=' .env; then
  KCSEC="$(grep '^KEYCLOAK_ADMIN_CLIENT_SECRET=' .env | head -1 | cut -d= -f2- | tr -d '\r\n"')"
else
  KCSEC=""
fi

# Skip-condition: empty / placeholder / too short to be meaningful.
case "$KCSEC" in
  ""|"changeme"|"your-secret-here"|"<"*">"|"\${"*"}")
    LIVE_VALUE_PRESENT="no"
    ;;
  *)
    if [ "${#KCSEC}" -lt 16 ]; then
      LIVE_VALUE_PRESENT="no-too-short"
    else
      LIVE_VALUE_PRESENT="yes"
    fi
    ;;
esac
```

If `LIVE_VALUE_PRESENT=yes`, run the precise scan:

```bash
# Pickaxe over all refs for the literal value. NEVER echo $KCSEC.
# The variable expands into git's argv directly; the audit DOES NOT print it.
git log --all -p -S "$KCSEC" -- '*' > /tmp/9iu-livevalue-hits.diff
LIVE_HIT_COUNT=$(grep -c '^commit ' /tmp/9iu-livevalue-hits.diff || echo 0)

# Cross-check: also pickaxe the SHA-256 (in case anyone ever stored a hash):
KCSEC_SHA16="$(printf '%s' "$KCSEC" | sha256sum | cut -c1-16)"
git log --all -p -S "$KCSEC_SHA16" -- '*' > /tmp/9iu-livevalue-sha-hits.diff || true

# Compute the redacted form for the report (4+4 rule):
KCSEC_REDACTED="$(printf '%s' "$KCSEC" | head -c 4)…$(printf '%s' "$KCSEC" | tail -c 4)"

# Unset to limit shell exposure.
unset KCSEC
```

Important: do NOT pipe `$KCSEC` into `tee`, `cat`, `>` redirection to a tracked file, or any
command whose output is captured into the chat. The only acceptable consumer is `git log -S`.
