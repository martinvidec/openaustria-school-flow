# Phase 8 — UI Design Contract

> Visual and interaction contract for Phase 8: Homework, Exams & Data Import. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification: FIFTH FRONTEND PHASE.** All design tokens (spacing, typography, base colors, semantic colors, subject palette, change indicator colors) are inherited verbatim from Phase 4/5/6/7 UI-SPECs. Phase 8 adds homework/exam badge indicators on timetable cells, a homework/exam assignment dialog, a multi-step data import wizard (Untis XML and CSV), column mapping UI, iCal subscription management, and copywriting within the established design system. No new color tokens are introduced.

## Spacing Scale

Declared values inherited from Phase 4 UI-SPEC (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, timetable badge icon-to-text gap, import status icon spacing |
| sm | 8px | Compact element spacing, homework/exam list item internal padding, import row gaps, column mapping field gap |
| md | 16px | Default element spacing, dialog form field gaps, import wizard step padding, homework detail card padding |
| lg | 24px | Section padding, import wizard panel padding, homework/exam dialog padding, iCal settings card padding |
| xl | 32px | Layout gaps, import wizard step-to-content gap, page-level header-to-content gap |
| 2xl | 48px | Major section breaks, page-level vertical padding |
| 3xl | 64px | Page-level spacing, content max-width gutters |

Exceptions (Phase 8 specific):
- Touch targets on mobile: minimum 44x44px (WCAG 2.5.5 AAA). Applies to homework/exam badge tap targets on timetable cells, import file drop zone, column mapping drag handles, iCal copy-URL button.
- Timetable cell badge area: 16x16px icon badges positioned in the top-right corner of the TimetableCell with 2px offset from edge. Multiple badges (homework + exam) stack horizontally with 2px gap.
- Import file drop zone: minimum 200px height for comfortable drag-and-drop on desktop and tablet.
- Column mapping table row height: 48px per row (accommodates source column label + dropdown selector at 44px touch target).
- Import progress bar height: 8px (consistent with standard shadcn progress bar sizing).

| Accent (10%) | `--color-primary` | hsl(221 83% 53%) | See reserved-for list below |

**Accent reserved for (exhaustive list for Phase 8):**
1. Primary CTA buttons: "Hausaufgabe erstellen", "Pruefung eintragen", "Import starten", "Importieren", "URL kopieren", "Token erneuern"
2. Active step indicator in import wizard (current step number/icon)

5. iCal URL copy-to-clipboard success icon flash (1s duration)
6. Homework badge icon fill on timetable cells (BookOpen icon, 16x16)

**Not permitted for:** generic hover states (use `--color-muted`), generic borders (use `--color-border`), timetable cell backgrounds (use subject color palette), decorative backgrounds, import file drop zone border (use `--color-border`, dashed).

| Warning | `--color-warning` | hsl(38 92% 50%) | Exam collision warning banner background (with 15% opacity), exam badge icon fill on timetable cells (ClipboardList icon, 16x16), import partial success (some rows skipped), duplicate row warning in import preview |
| Destructive | `--color-destructive` | hsl(0 84% 60%) | Delete homework confirmation, delete exam confirmation, import row validation error indicator, import failure toast, iCal token revoke confirmation |

### Timetable Badge Colors (Phase 8 specific)

| Badge Type | Icon | Background | Icon Color | Usage |
|------------|------|-----------|------------|-------|
| Homework | BookOpen (Lucide) | transparent (overlay on cell) | `--color-primary` hsl(221 83% 53%) | Small badge in top-right corner of timetable cell indicating homework is assigned for this lesson |
| Exam | ClipboardList (Lucide) | transparent (overlay on cell) | `--color-warning` hsl(38 92% 50%) | Small badge in top-right corner of timetable cell indicating an exam is scheduled. Warning color for visual urgency differentiation from homework. |

**Badge rendering rule:** Badges are 16x16px Lucide icons positioned absolutely in the top-right of the TimetableCell container. When both homework and exam exist for the same lesson, homework badge is leftmost, exam badge is rightmost, separated by 2px horizontal gap. Badges do not alter the cell's subject color background. On hover/tap, a tooltip shows the homework title/due date or exam title/date.

### Import State Colors (Phase 8 specific)

| State | Background | Text/Icon | Usage |
|-------|-----------|-----------|-------|
| Pending (queued) | `--color-muted` | `--color-muted-foreground` | Import job in queue, not yet started |
| Processing | `--color-primary` at 15% opacity | `--color-primary` | Import job actively running, progress bar filling |
| Success | `--color-success` at 15% opacity | `--color-success` | Import job completed with all rows imported |
| Partial | `--color-warning` at 15% opacity | `--color-warning` | Import completed but some rows skipped or had warnings |
| Failed | `--color-destructive` at 15% opacity | `--color-destructive` | Import job failed entirely |

## Interaction Contracts

### Homework Assignment (D-01, HW-01)

| Interaction | Behavior |
|-------------|----------|
| Entry point | Teacher clicks a lesson in the classbook detail page. "Hausaufgabe erstellen" button appears in the lesson detail panel (alongside existing Anwesenheit, Inhalt tabs). |
| Create dialog | Opens HomeworkDialog in create mode. Title (required), description (optional), due date (required, defaults to next occurrence of this lesson), class subject pre-filled from lesson context. |
| Submit | POST to homework API. Success toast: "Hausaufgabe erstellt". Dialog closes. Timetable cells for this lesson display HomeworkBadge. Notification sent to students and parents of the class (D-11). |
| Edit | Teacher clicks homework in HomeworkExamList or via timetable cell badge popover "Bearbeiten" link. HomeworkDialog opens in edit mode with pre-filled values. |
| Delete | Dropdown action "Loeschen" on homework list item. Confirmation dialog with copy: "Hausaufgabe loeschen: Moechten Sie die Hausaufgabe '{title}' wirklich loeschen?" On confirm: DELETE request, success toast, badge removed from timetable cell. |
| Student/parent view | Students and parents see HomeworkBadge on timetable cells. Clicking badge opens read-only popover with title, description, due date. No edit/delete actions. |
| Validation | Title required (min 1 character). Due date required, must be today or future. |

### Exam Scheduling (D-02, D-03, HW-02)

| Interaction | Behavior |
|-------------|----------|
| Entry point | Teacher clicks "Pruefung eintragen" button on a class page, in the classbook lesson detail, or from the HomeworkExamList "Pruefungen" tab. |
| Create dialog | Opens ExamDialog in create mode. Title (required), date (required), class (pre-filled from context), subject (pre-filled from context), duration in minutes (optional), description (optional). |
| Collision detection | On date change, frontend queries existing exams for the same class on the selected date. If collision found, ExamCollisionWarning banner appears inline below the date field. Warning text: "Achtung: Am {date} ist bereits eine Pruefung fuer diese Klasse eingetragen ({existingExamTitle}). Trotzdem eintragen?" This is a soft warning -- teacher can proceed with "Trotzdem eintragen" or change the date. (D-03) |
| Submit | POST to exam API. Success toast: "Pruefung eingetragen". Dialog closes. Timetable cells on the exam date display ExamBadge. Notification sent to students and parents (D-11). |
| Edit | Teacher clicks exam in HomeworkExamList or via timetable cell badge popover. ExamDialog in edit mode. |
| Delete | Confirmation dialog. On confirm: DELETE request, success toast, badge removed. |
| Student/parent view | Students and parents see ExamBadge (warning color) on timetable cells. Clicking badge opens read-only popover with exam title, date, subject, duration. In-app notification received when exam is scheduled (D-11). |

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Timetable badges | `aria-label="Hausaufgabe: {title}"` / `aria-label="Pruefung: {title}"` on badge icons. Popover content accessible via keyboard (Enter/Space to open, Escape to close). |
| Import wizard | Step indicator uses `aria-current="step"` on active step. Form fields have associated labels. Error messages linked via `aria-describedby`. Progress bar uses `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`. |
| File drop zone | `role="button"` with `aria-label="Datei auswaehlen"`. Keyboard accessible via Enter/Space. Drag events supplemented by click-to-browse fallback. |
| Column mapping | Select dropdowns keyboard navigable. Required fields announced via `aria-required="true"`. |
| Exam collision warning | `role="alert"` with `aria-live="polite"` so screen readers announce the collision when it appears. |
| Import status | Status badges use `aria-label` describing the status in full text (not just color). |
