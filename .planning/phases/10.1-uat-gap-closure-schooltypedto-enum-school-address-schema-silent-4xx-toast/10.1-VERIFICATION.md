### Human Verification Required

The automated layer (unit specs, file inspection) confirms all structural invariants hold. Five behaviors require browser-level confirmation before Phase 10 UAT (Plan 10-06 Task 2) can be declared complete:

#### 1. Silent-4xx toast — Stammdaten tab

**Test:** Log in as admin. Open an existing school's Stammdaten tab. In DevTools, block or intercept the PUT endpoint to force a 4xx response. Submit the form.
**Expected:** A red Sonner toast appears with the error message. No green "Aenderungen gespeichert." toast. On reload, no data change is visible.
**Why human:** The Vitest jsdom environment proves the hook fires `toast.error` instead of `toast.success`. It cannot prove Sonner's visual rendering, toast stacking (both toasts firing but green appearing first), or the race between optimistic UI updates and the error handler.

#### 2. Silent-4xx toast — OptionsTab A/B toggle

**Test:** Toggle the A/B-Wochen switch while the network tab is set to Offline or the API is stopped.
**Expected:** Red "Speichern fehlgeschlagen" toast. Switch returns to unchecked. No green "Option gespeichert." toast.
**Why human:** Same as above — visual rendering and network failure modes cannot be reliably simulated in jsdom.

#### 3. Silent-4xx toast — Schuljahre tab

**Test:** Attempt to create a school year with an invalid date range (e.g., endDate before startDate) to force a backend 4xx.
**Expected:** Red error toast with backend message. Dialog stays open or closes with error state. No green toast.
**Why human:** Same as above.

#### 4. Address round-trip via HTTP

**Test:** Run `curl -sS -X POST http://localhost:3000/api/v1/schools -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" -d '{"name":"Round-trip Test","schoolType":"AHS","address":{"street":"Testgasse 1","zip":"1010","city":"Wien"}}'` and then `curl -sS http://localhost:3000/api/v1/schools/$SID -H "Authorization: Bearer $TOKEN" | jq '.address'`.
**Expected:** GET response contains `{"street":"Testgasse 1","zip":"1010","city":"Wien"}` as a JSON object, not a stringified value.
**Why human:** `school.service.spec.ts` mocks Prisma and proves `dto.address` is passed as an object to `prisma.school.create`. It cannot prove the full NestJS serialization pipeline (class-transformer `@Expose`, `plainToInstance`, `SerializeInterceptor` if any) returns the correct shape through the HTTP response.

#### 5. Stammdaten form pre-fill from repaired address

**Test:** Start dev stack after running `pnpm --filter @schoolflow/api exec prisma db seed`. Open the seeded school `BG/BRG Musterstadt` in the Stammdaten tab. Observe the Strasse, PLZ, and Ort fields.
**Expected:** Fields show "Schulstrasse 1", "1010", "Wien" respectively. No "[object Object]", no empty fields.
**Why human:** Requires confirming the frontend's SchoolDetailsTab correctly maps `school.address.street`, `school.address.zip`, `school.address.city` to the three form inputs — a visual/UI wiring check that no unit test currently covers.
