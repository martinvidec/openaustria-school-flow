<decision-record>
**D-A Address column shape: JSON (not normalized columns).**

Two options in UAT-FINDINGS §Bug 2:
  (a) 3 separate columns: `addressStreet`, `addressZip`, `addressCity`
  (b) Single `Json? @db.JsonB` column holding `{street, zip, city}`

This plan takes **(b) — JSON column**. Rationale:
  - Shared DTO + UI already use the nested object (packages/shared locked). (b) requires
    ZERO wire-format changes — CreateSchoolDto accepts the object as-is, service passes
    it to Prisma, response DTO returns the object unchanged.
  - (a) requires a mapping layer on both sides of the API boundary (flatten on write,
    re-assemble on read) — more surface area, more places to drift.
  - Query ergonomics matters only if we ever filter schools by address fragments.
    Phase 10-16 roadmap shows no such filter requirement. If one arises later,
    `jsonb_path_ops` + GIN index on `address` solves it; or Phase N+1 adds the 3
    scalar columns on top of the JSON.
  - The corrupt row repair is simpler for JSON: `UPDATE ... SET address = NULL
    WHERE address = '[object Object]'` — no column-splitting gymnastics.

**D-B Corrupt row repair = reset to NULL.**

The seeded row has `address = '[object Object]'` — no recoverable structure. Setting
it back to NULL is honest and idempotent. The seed script (see Task 3) will then
re-populate it with the proper `{street,zip,city}` object on the next `prisma migrate
reset + seed`, so dev-environment parity is preserved.

Alternative: parse the string with a regex. REJECTED — `'[object Object]'` carries
zero user data; parsing would at best fabricate placeholder values.
</decision-record>

```sql
-- Phase 10.1 Bug 2: convert schools.address from text to jsonb, repairing the one
-- known-corrupt row (seed-school-bgbrg-musterstadt stored '[object Object]' from an
-- accidental toString() on the frontend). Pre-clear any non-JSON strings to NULL so

-- row and any future text row someone might have manually inserted).
UPDATE "schools"
SET "address" = NULL
WHERE "address" IS NOT NULL
  AND NOT (
    "address" ~ '^\s*\{'   -- starts with an object brace
  );

-- Step 2: convert the column type. Use USING to coerce the surviving text values
-- through jsonb::text -- any remaining row must already contain parseable JSON.
ALTER TABLE "schools"
  ALTER COLUMN "address" TYPE jsonb USING
    CASE
      WHEN "address" IS NULL THEN NULL
      ELSE "address"::jsonb
    END;

-- Step 3: explicit belt-and-braces repair for the known seed row in case pre-existing
-- dev DBs skipped Step 1 (e.g. the row was NULL already but the column type change ran).
UPDATE "schools"
SET "address" = NULL

  AND (

  );
```
