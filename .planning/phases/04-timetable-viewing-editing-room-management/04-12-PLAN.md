  <action>
    **Problem:** NestJS with Fastify adapter does not automatically bind Socket.IO to the underlying HTTP server the same way it does with Express. The `@nestjs/platform-socket.io` package provides an `IoAdapter` that must be explicitly registered when using Fastify, otherwise WebSocket connections may fail because Socket.IO cannot intercept the HTTP upgrade request.

    **Fix in apps/api/src/main.ts:**

    1. Add import for IoAdapter at the top of the file:
    ```typescript
    import { IoAdapter } from '@nestjs/platform-socket.io';
    ```

    2. After the `app.enableCors(...)` call (line 26) and before the global pipes, add:
    ```typescript
    // Socket.IO adapter for Fastify compatibility
    // Required because Fastify does not share Express's built-in HTTP upgrade handling.
    // Without this, @WebSocketGateway decorators may fail to bind Socket.IO to the server.
    app.useWebSocketAdapter(new IoAdapter(app));
    ```

    The complete bootstrap function should have this order:
    1. `NestFactory.create` with FastifyAdapter
    2. `app.setGlobalPrefix('api/v1')`
    3. `app.enableCors(...)`
    4. `app.useWebSocketAdapter(new IoAdapter(app))` -- NEW
    5. `app.useGlobalPipes(createValidationPipe())`
    6. `app.useGlobalFilters(new ProblemDetailFilter())`
    7. Swagger setup
    8. `app.listen(...)`

    **Verify Vite proxy is correct (read-only check):**

    The Vite config at `apps/web/vite.config.ts` line 37 already has:
    ```typescript
    '/socket.io': { target: 'http://localhost:3000', ws: true },
    ```

    This is correct. Socket.IO uses `/socket.io/` as the transport path for all namespaces (the namespace is in the query string, not the URL path). The `ws: true` flag enables WebSocket upgrade proxying. No changes needed to Vite config.

    **Verify frontend Socket.IO client is correct (read-only check):**

    The `apps/web/src/lib/socket.ts` connects to `${API_URL}/timetable` with `transports: ['websocket', 'polling']`. When `API_URL` is empty (default dev), this connects to the same origin, which Vite proxies via the `/socket.io` rule. This is correct. No changes needed.
