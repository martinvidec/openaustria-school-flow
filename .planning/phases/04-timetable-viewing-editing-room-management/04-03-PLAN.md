    **4. Create apps/api/src/modules/timetable/timetable-edit.service.ts:**
    New service (separate from TimetableService to keep single responsibility) with PrismaService injection:

    **validateMove(schoolId, dto: ValidateMoveDto) -> MoveValidationResponseDto:**
    1. Find the lesson by ID with its run (must be active run)
    2. Check hard constraints:
       a. Teacher clash: Is this teacher already teaching at targetDay + targetPeriod in the same run? Query: `prisma.timetableLesson.findFirst({ where: { runId, teacherId: lesson.teacherId, dayOfWeek: targetDay, periodNumber: targetPeriod, weekType compatible, id: { not: lessonId } } })`
       b. Room clash: Is the target room (lesson's current room or targetRoomId) occupied? Same pattern with roomId.
       c. Student group clash: Is the classSubject's class already scheduled at targetDay + targetPeriod? Query through ClassSubject -> classId.
    3. Check soft constraints:
       a. Max lessons per day: Count lessons for this teacher on targetDay. If >= 8, add warning "Lehrer hat bereits 8 Stunden an diesem Tag".
       b. Subject doubling: Is the same subject already scheduled at targetPeriod for the same class? Add warning "Fach ist bereits in dieser Stunde eingeplant".
    4. Return { valid: hardViolations.length === 0, hardViolations, softWarnings }

    **moveLesson(schoolId, dto: ValidateMoveDto, userId: string) -> TimetableViewLessonDto:**
    1. Call validateMove first. If hardViolations.length > 0, throw BadRequestException with violations.
    2. Save previous state as JSON: { dayOfWeek, periodNumber, roomId, weekType }
    3. Update TimetableLesson: set dayOfWeek, periodNumber, roomId (if targetRoomId provided), isManualEdit = true, editedBy = userId, editedAt = new Date()
    4. Create TimetableLessonEdit record: { lessonId, runId, editedBy: userId, editAction: 'move', previousState, newState: { dayOfWeek: targetDay, periodNumber: targetPeriod, roomId } }
    5. Return updated lesson as TimetableViewLessonDto (with joined data)
    Both operations in a Prisma transaction.

    **getEditHistory(runId: string) -> LessonEditHistoryDto[]:**
    Query TimetableLessonEdit where runId, ordered by createdAt DESC. Map to DTO.

    **revertToEdit(runId: string, editId: string, userId: string) -> void:**
    1. Find the target edit record
    2. Find all edits AFTER this edit (createdAt > target.createdAt) for this run
    3. For each later edit (in reverse chronological order), restore the lesson's previousState
    4. Create a new TimetableLessonEdit with editAction: 'revert' recording the revert operation
    5. All in a Prisma transaction
