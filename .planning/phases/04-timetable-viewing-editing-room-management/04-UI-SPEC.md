# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Timetable Viewing, Editing & Room Management. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification: FIRST FRONTEND PHASE.** This phase scaffolds the entire React SPA (`apps/web/`) and establishes all UI patterns for the project. All design tokens declared here carry forward to Phases 5-9.

## Spacing Scale

Declared values (multiples of 4, inherited from Phase 1 UI-SPEC):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, timetable cell internal icon spacing |
| sm | 8px | Compact element spacing, timetable cell line spacing (subject/teacher/room), form field internal padding |
| md | 16px | Default element spacing, card padding, timetable grid gap between day columns |
| lg | 24px | Section padding, dialog padding, sidebar section gaps |
| xl | 32px | Layout gaps between major sections (grid and controls, sidebar and content) |
| 2xl | 48px | Major section breaks, page-level vertical padding |
| 3xl | 64px | Page-level spacing, content max-width gutters |

Exceptions:
- Touch targets on mobile: minimum 44px (WCAG 2.5.5 AAA). Applies to timetable cells in day view, A/B week tabs, day/week toggle, perspective dropdown, room booking slots.
- Timetable grid row height: 56px per regular period row (accommodates 3 lines of text at 14px + padding). Break rows: 24px height.
- Doppelstunde merged cell: 120px (56px * 2 + 8px gap between rows).

**Timetable cell-specific typography:**

| Element | Size | Weight | Line Height |
|---------|------|--------|-------------|

## Interaction Contracts

### Timetable Grid (VIEW-01, VIEW-02, VIEW-03)

| Interaction | Behavior |
|-------------|----------|
| Default view (desktop) | Week view showing 5 day columns (Mo-Fr) + period label column |
| Default view (mobile) | Day view showing today only. Swipe left/right to navigate days |
| Day/week toggle | Click [Tag] or [Woche] to switch. Persisted in Zustand store |
| A/B week tabs | Click [A-Woche] or [B-Woche]. Only visible when school `weekMode !== 'REGULAR'`. Default = current week |
| Perspective selector (admin) | Dropdown with 3 groups: Lehrer (list), Klassen (list), Raeume (list). Search-enabled. Selected item loads that timetable |
| Perspective selector (teacher) | Hidden. Automatically shows "Mein Stundenplan" |
| Perspective selector (student/parent) | Hidden. Automatically shows class timetable |
| Period hover | Tooltip shows start time - end time (e.g., "08:00 - 08:50") |
| Cell click (non-admin) | No action. Read-only |
| Cell click (admin, edit mode) | Select lesson for detail panel or start drag |
| Break row | Reduced height (24px), muted background, non-interactive. Label shows break name if configured |
| Doppelstunde | Single merged cell spanning 2 period rows. Same content, taller cell |
| Empty slot | Muted background, no content. In admin edit mode: droppable target |

### Drag-and-Drop Editing (TIME-08, D-07, D-08, D-09, D-10)

| Interaction | Behavior |
|-------------|----------|
| Enter edit mode | Admin clicks "Bearbeiten" button to toggle edit mode on the timetable |
| Drag start | Lesson cell becomes draggable. Original position shows dashed blue outline. Ghost overlay follows cursor |
| Drag over slot | Debounced (200ms) validation call to `POST /api/v1/schools/:schoolId/timetable/validate-move`. Slot shows green/yellow/red overlay |
| Drop on valid slot | Lesson moves to new position. `PATCH` request to update lesson. Toast: "Stunde verschoben" |
| Drop on hard violation | Drop rejected. Slot flashes red. Tooltip shows violation: "Lehrerkonflikt: {teacherName} unterrichtet bereits in Periode {N}" |
| Drop on soft warning | Drop allowed. Yellow indicator persists on cell. Toast: "Stunde verschoben (Hinweis: {warningText})" |
| Drop on break row | Drop rejected. Break rows are never valid drop targets |
| Cancel drag | Press Escape or drop outside grid. Lesson returns to original position |
| Keyboard DnD | Tab to select lesson, Space to pick up, Arrow keys to navigate slots, Space to drop. Full @dnd-kit keyboard sensor support |
| Edit history | Every move persisted via TimetableLessonEdit model. Admin can view history in side panel and revert to any prior state |

### Room Booking (ROOM-03, D-13, D-14)

| Interaction | Behavior |
|-------------|----------|
| View room availability | Navigate to Raeume page. Grid shows rooms (rows) x periods (columns) for selected day |
| Filter rooms | Filter by: room type (dropdown), minimum capacity (number input), equipment tags (multi-select) |
| Book a room | Click green (free) slot. Dialog opens: room name, day, period pre-filled. Teacher enters purpose (optional). Click "Raum buchen" |
| Cancel booking | Click own booking in grid. Dialog: "Buchung stornieren?" with "Stornieren" button |
| View room timetable | Click room name in grid header. Shows that room's timetable (solver lessons + ad-hoc bookings) |

### Resource Management (ROOM-04, D-15)

| Interaction | Behavior |
|-------------|----------|
| View resources | Navigate to admin Ressourcen page. Table shows name, type, quantity, description |
| Add resource | Click "Ressource hinzufuegen". Dialog with name, type (select), quantity (number), description (textarea) |
| Edit resource | Click edit icon on row. Same dialog, pre-filled |
| Delete resource | Click delete icon. Confirmation: "Ressource '{name}' loeschen? Alle zugehoerigen Buchungen werden ebenfalls geloescht." |
| Book resource | From room booking dialog, optional resource select. Links resource to room+period combination |

### Export (VIEW-06)

| Interaction | Behavior |
|-------------|----------|
| Open export menu | Click export button (download icon + "Exportieren") in timetable header |
| Export as PDF | Click "Als PDF exportieren". Server generates PDF. Browser downloads `stundenplan-{perspective}-{date}.pdf` |
| Export as iCal | Click "Als iCal exportieren". Server generates .ics file. Browser downloads `stundenplan.ics` |
| Export scope | Exports the currently viewed timetable (respects perspective selector and A/B week selection) |

### Real-Time Updates (VIEW-04, ROOM-05, D-12, D-16)

| Event | Client Behavior |
|-------|-----------------|
| `timetable:changed` | TanStack Query invalidation for `['timetable', schoolId]`. Toast: "Stundenplan aktualisiert: {N} Aenderungen" |
| `timetable:cancelled` | Same invalidation. Affected cells show red border + "Entfall" badge |
| `timetable:room-swap` | Invalidate timetable + room availability queries. Toast: "Raumaenderung durchgefuehrt" |
| `timetable:substitution` | Invalidate timetable queries. Affected cells show orange border + new teacher name |
| Connection lost | Sonner toast: "Verbindung unterbrochen. Versuche erneut zu verbinden..." Auto-reconnect via Socket.IO (up to 10 attempts, 1s delay) |
| Reconnected | Toast dismissed. Silent background refetch of all active queries |

**Copywriting conventions (from Phase 1, apply here):**
- Formal "Sie" address (standard in Austrian school administration)
- No exclamation marks in error messages
- Error messages always include a next-step action
- Button labels use imperative verb + noun
- Confirmation dialogs for destructive actions always state the consequence explicitly
- Toast messages are concise (under 60 characters) and disappear after 4 seconds
- Toast messages for WebSocket updates disappear after 3 seconds
