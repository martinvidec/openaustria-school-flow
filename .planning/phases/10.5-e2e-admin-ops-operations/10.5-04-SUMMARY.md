## Accomplishments

- **D-10 (solve endpoint):** contract pinned in DISCOVERY.md — `POST /timetable/solve` returns 202 with `{runId, status:'QUEUED'}`, body `@Min(30)/@Max(600)`.
- **D-11 (sidecar health):** `GET :8081/health → {"status":"UP"}` confirmed + wired into `global-setup.ts` at parity with the docker-compose probe. RESEARCH.md Pitfall 1 (CONTEXT.md D-11 mis-documented URL :8080) fixed.
- **D-12 (Aktivieren button):** new button in `solver.tsx` inside the "Letztes Ergebnis" Card, wired to `POST /timetable/runs/:runId/activate` + green toast `Stundenplan aktiviert` + `navigate('/timetable')` on success. Red toast `Aktivierung fehlgeschlagen (HTTP ${status})` on failure; button stays enabled for retry (no early-return redirect).
- **D-13 (SOLVER-01 happy spec):** authored — solve via `request.post()`, wait for `getByCardTitle('Letztes Ergebnis')` (Pattern 6), click Aktivieren, assert green toast + URL regex `/timetable` + grid visibility.
- **D-13 (SOLVER-02 activate-error spec):** authored — real solve (populates lastResult) + `page.route('**/activate')` 500 mock, assert red toast stem via regex (Pitfall 8), silent-4xx-guard (no green toast), button still enabled, no navigation off `/admin/solver`.
- **D-14 (allowed prod-code set):** respected — only `solver.tsx` and `global-setup.ts` touched. No `apps/solver/` edits despite finding bugs there (logged as deferred-items §6-§7 instead).
- **Opt-in E2E gate:** `test.skip(!RUN_SOLVER, ...)` at describe level — spec is checked in as executable documentation; flipping `E2E_RUN_SOLVER=1` + the §6 workaround runs it locally.

## Blockers discovered (logged, not fixed)

- **§6 (Timefold Enterprise):** `move-thread-count=AUTO` requires commercial Enterprise Edition. Community Edition throws `IllegalStateException` at solver boot. Workaround: `QUARKUS_TIMEFOLD_SOLVER_MOVE_THREAD_COUNT=NONE`.
- **§7 (duplicate planningIds):** Solver domain input creates `SolverTimeslot` rows whose `planningId` UUID collides across weekdays. Timefold rejects with `"Working objects must be unique"`. Fix belongs to `apps/solver/src/main/java/...` (compose a deterministic `{dayOfWeek}-{periodNumber}-{weekType}` planningId).

Both bugs are outside the plan's D-14 allowed path set. Logging them as deferred-items matches the 10.1+ precedent for "found while testing, fix belongs to a different tranche."
