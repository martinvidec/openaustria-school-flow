## /admin/resources DOM contract

### Add-Button
- Accessible name (exact): `Ressource hinzufuegen`  (ASCII `ue`, not `ü` — `resources.tsx:147`)
- Locator: `page.getByRole('button', { name: 'Ressource hinzufuegen' })`

### Add/Edit form fields (all inside a shadcn `<Dialog>`; no `<form role="form">`)
- Name field label: `Name` (exact, `<label htmlFor="resource-name">Name</label>`, `resources.tsx:218`)
  - Input is `<input id="resource-name" type="text" required>` — `page.getByLabel('Name')` resolves.
- Typ field label: `Typ` (exact, `<label htmlFor="resource-type">Typ</label>`, `resources.tsx:237`)
  - Underlying widget is a shadcn `<Select>` (Radix) rendered as `<button role="combobox" id="resource-type">`.
  - `page.getByLabel('Typ')` resolves to that combobox button. Use:
    ```
    await page.getByLabel('Typ').click();
    await page.getByRole('option', { name: '<enum value>' }).click();
    ```
- Menge field label: `Menge` (exact, `<label htmlFor="resource-quantity">Menge</label>`, `resources.tsx:266`)
  - Input is `<input type="number" min=1>` — `page.getByLabel('Menge')` resolves.
- Beschreibung field label: `Beschreibung (optional)` (exact, `resources.tsx:287`) — optional; specs do not fill.
- submit button name (exact): `Speichern` (`resources.tsx:315`); pending label is `Wird gespeichert...` so specs must assert on `Speichern` before click.
- Cancel button (exact): `Abbrechen` (`resources.tsx:312`).

### Dialog titles
- Add mode: `Ressource hinzufuegen` (exact, `resources.tsx:211`)
- Edit mode: `Ressource bearbeiten` (exact, `resources.tsx:210`)
- Delete confirm dialog title: `Ressource loeschen` (exact, `ResourceList.tsx:129`; ASCII `oe`)
- Delete confirm destructive button: `Loeschen` (exact, `ResourceList.tsx:150`); pending label `Wird geloescht...`.

### List row structure (`ResourceList.tsx`)
- Rendered as HTML `<table>` with `<thead>` columns `Name / Typ / Menge / Beschreibung / Aktionen`
  and one `<tr>` per resource (`ResourceList.tsx:81-117`).
- Each row exposes:
  - The resource name as text in the first `<td>`.
  - An edit button with `aria-label="{name} bearbeiten"` (e.g. `E2E-RES-01-... bearbeiten`) — `ResourceList.tsx:102`.
  - A delete button with `aria-label="{name} loeschen"` (ASCII `oe`, e.g. `E2E-RES-01-... loeschen`) — `ResourceList.tsx:110`.
- Preferred (most stable) locators — scope directly by the per-row accessible name:
  - `page.getByRole('button', { name: `${name} bearbeiten` })`
  - `page.getByRole('button', { name: `${name} loeschen` })`
- Row-scoped locator for assertions that the row exists or disappears:
  - `page.getByRole('row', { name: new RegExp(name) })`
  (Playwright matches `<tr>` accessible name from cell text.)

### Delete flow
1. Click `<row>{name} loeschen` button → opens confirm Dialog with title `Ressource loeschen`.
2. Click destructive `Loeschen` inside the confirm dialog (scope with `.last()` because the row's
   icon-only button also matches `/[Ll]oeschen/`; the dialog's textual `Loeschen` renders later in
   the DOM — mirrors the 10.2-03 SCHOOL-05 pattern at `schuljahre.spec.ts:125`).

### resourceType enum values (resourceType enum — from frontend RESOURCE_TYPES, `resources.tsx:34-39`)
The backend DTO (`resource.dto.ts:10-12`) validates `resourceType` only as non-empty `string`
(no `@IsEnum`) — any free-form string is accepted. The admin UI offers four options:

- `Tablet-Wagen`
- `Laborgeraet`
- `Beamer`
- `Sonstiges`

Spec target for happy-paths: `Beamer` (matches CONTEXT.md D-07 example: `resourceType=Beamer`).
The Radix `<SelectItem value="Beamer">Beamer</SelectItem>` renders option text `Beamer`, so
`page.getByRole('option', { name: 'Beamer' })` resolves unambiguously.

### Endpoint shape (VERIFIED — `resource.controller.ts`)
- `POST   /api/v1/schools/:schoolId/resources`         — create (`CreateResourceDto`: name, resourceType, quantity?, description?)
- `GET    /api/v1/schools/:schoolId/resources`         — list (for afterEach cleanup filter)

### Notes for the mobile spec
- `DialogContent` uses `h-[100dvh] sm:h-auto sm:max-w-lg max-w-full` (`resources.tsx:206`) —
  full-screen dialog on mobile by design, so form inputs remain reachable at 375px.
- `ResourceList` keeps its `<table>` at every breakpoint; the page wraps it in
  `overflow-x-auto` (`resources.tsx:193`) for horizontal scroll on narrow viewports.
  Row-level selectors (`getByRole('button', { name: '... loeschen' })`) remain valid on mobile.
- `playwright.config.ts:41-43` — `mobile-375` project runs files matching `*.mobile.spec.ts`,
  viewport 375×812 (iPhone 13 UA). `testMatch` excludes `*.mobile.spec.ts` from desktop project.
