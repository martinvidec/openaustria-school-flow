## 3. Deterministic free slot

**Blocker discovered:** the `seed-school-bgbrg-musterstadt` seed currently ships **no rooms and no time-grid periods**. A raw `GET /rooms/availability?dayOfWeek=MONDAY` returns `[]`. This blocks every booking test.

**Resolution (Rule 3 — auto-fix blocking):** the spec self-provisions test data in `beforeAll` via the admin token — seeds a time-grid (2 non-break periods + MONDAY-FRIDAY school days) and a throwaway room named `E2E-ROOM-BOOK01-<timestamp>`. Cleanup in `afterAll` deletes the throwaway room (which cascades its bookings). No seed.ts mutation required; the approach mirrors 10.4-01 SCHOOL-03's throwaway year pattern.

**Probed endpoints (all live at 2026-04-22):**

| Method | URL | Status | Purpose |
| --- | --- | --- | --- |
| PUT | `/schools/:schoolId/time-grid` | 200 | Seeds `{periodNumber, startTime, endTime, isBreak, durationMin}` + `schoolDays` |
| POST | `/schools/:schoolId/rooms` | 201 | Creates throwaway room (`name`, `roomType=KLASSENZIMMER`, `capacity`) |
| GET | `/schools/:schoolId/rooms/availability?dayOfWeek=MONDAY` | 200 | Returns one slot per room × period |
| POST | `/schools/:schoolId/rooms/bookings` | 201 | Creates booking |
| POST (dup) | `/schools/:schoolId/rooms/bookings` | **409** | Hits `ConflictException` (verified above) |
| DELETE | `/schools/:schoolId/rooms/bookings/:bookingId` | 204 | Cancels booking |
| DELETE | `/schools/:schoolId/rooms/:roomId` | 204 | Deletes room |

**Locator for free-slot cells:** `RoomAvailabilityGrid.tsx:214` gives available cells

```tsx
aria-label={`${roomName}, Stunde ${slot.periodNumber} - Frei`}
```

So `page.getByRole('button', { name: /E2E-ROOM-BOOK01-\d+, Stunde 1 - Frei/ })` is the deterministic "click the free slot" locator — narrow by the throwaway room's name so a seed-noise change never steals the match.

**Chosen slot shape per test:**

- **ROOM-BOOK-01 (Happy):** period 1, MONDAY, throwaway `E2E-ROOM-BOOK01-<ts>`
- **ROOM-BOOK-02 (Conflict):** period 2, MONDAY, same throwaway room; first booking via API, second via UI (Research Pattern 3)

Using **different period numbers per test** avoids the parallel-worker race Pitfall 5 flagged, even though both tests target the same throwaway room.

## 4. API cleanup path

- `DELETE /schools/:schoolId/rooms/bookings/:bookingId` **exists** (`apps/api/src/modules/room/room.controller.ts:125-138`) and returns 204. Verified live.
- `DELETE /schools/:schoolId/rooms/:id` also exists (line 102) and returns 204. Deleting the throwaway room cascades the bookings (Prisma schema has the FK on RoomBooking).

**afterEach strategy:** iterate bookings on the throwaway room via `GET /rooms/availability` (filters to `purpose: "E2E-BOOK-*"`), DELETE each by bookingId. Room itself lives across tests inside the describe block and is torn down in `afterAll`.

**No Prisma-direct fixture needed** — every cleanup path goes through the API with the admin token.
