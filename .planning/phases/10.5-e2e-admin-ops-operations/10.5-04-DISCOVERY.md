## solve endpoint

`POST /api/v1/schools/:schoolId/timetable/solve` with admin Bearer token + body `{"maxSolveSeconds":30}`:

- Status: **HTTP 202 Accepted**
- Body:

  ```json
  {"runId":"eff04058-554e-4530-89b9-23350851f6e7","status":"QUEUED"}
  ```

- runId path in response: `body.runId` (string UUID).

`StartSolveDto` (`apps/api/src/modules/timetable/dto/solve-request.dto.ts`) validates `maxSolveSeconds` with `@Min(30)` / `@Max(600)` â€” 30 is the minimum allowed, matching D-10. Default when omitted is `300`.

## activate endpoint

`POST /api/v1/schools/:schoolId/timetable/runs/:runId/activate` (timetable.controller.ts:121):

- Decorators: `@Post('runs/:runId/activate')`, `@HttpCode(HttpStatus.OK)`, `@CheckPermissions({ action: 'update', subject: 'timetable' })`
- Auth: JWT bearer + CASL permission `update:timetable` (admin + schulleitung satisfy via manage-all seed).
- Returns on success: `{ runId, activated: true }` with HTTP 200.

**Invalid runId path** (UUID not in DB), e.g. `00000000-0000-0000-0000-000000000000`:

- Status: **HTTP 404 Not Found**
- Body (`application/problem+json`):

  ```json
  {
    "type":"https://httpstatuses.com/404",
    "title":"Not Found",
    "status":404,
    "detail":"Die angeforderte Ressource wurde nicht gefunden.",
    "instance":"/api/v1/schools/.../runs/00000000-.../activate",
    "traceId":"req-m3"
  }
  ```

This confirms RESEARCH.md Pitfall 8: an invalid runId gives 404, not 500 â€” so SOLVER-02's mocked-500 scenario must assert on the error-toast **stem** (`Aktivierung fehlgeschlagen`) and not couple to the `(HTTP 500)` vs `(HTTP 404)` suffix produced by `solver.tsx` handleActivate's `new Error('HTTP ${res.status}')`.

**Valid runId path:** not covered in this discovery pass (the SOLVER-01 spec authored in Task 4 exercises the live path end-to-end with a real solved run; capturing a live-populated runId here would duplicate Task 4 work).
