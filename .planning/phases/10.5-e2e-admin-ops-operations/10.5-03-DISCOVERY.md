# 10.5-03 Discovery — ImportWizard DOM contract

Captured 2026-04-22 during Plan 10.5-03 finalization. Read-only inspection of
production source (D-23: no prod-code touches).

## Step 1 — ImportFileUpload

- File input: hidden `<input type="file" accept=".xml,.csv,.txt">` at
  `ImportFileUpload.tsx:119-125`. Triggered by click on the drop zone
  (`role="button" aria-label="Datei auswaehlen"`).
- Playwright contract: `page.locator('input[type="file"]').setInputFiles(path)`
  works directly against the hidden input — no need to click the drop zone.
- **No entity-type or file-type radio selector.** Type is auto-detected from
  file extension (`detectFileType()` at `ImportFileUpload.tsx:8-14`): `.xml` →
  `UNTIS_XML`, `.csv` → `CSV`, `.txt` → `UNTIS_DIF`.
- On success, `handleFileSelect` fires `useUploadImport.mutate` → backend returns
  `parseResult` → wizard auto-advances to Step 2 (`ImportWizard.tsx:101-108`).

## Step 2 — Branches by detectedType

### CSV branch (ImportColumnMapper)

- Desktop table rows (`tbody tr`), one per CSV header. Each row has:
  - `<td>` with header text (e.g. `firstName`)
  - `<td>` with 3-row sample preview
  - `<td>` with a `<Select>` (shadcn/Radix) — trigger has `role="combobox"`.
- Initial value for every header: `'__skip__'` (label `"Ueberspringen"`).
- Target field option labels (from `ImportWizard.TARGET_FIELDS.STUDENTS`):
  - `firstName` → `"Vorname"`
  - `lastName` → `"Nachname *"` (required — asterisk in label)
  - `email` → `"E-Mail"`
  - `class` → `"Klasse"`
- Conflict-mode Select at bottom ("Bei Duplikaten"): default `SKIP` label
  `"Ueberspringen"`. Not required to change for happy-path.
- **`"Weiter"` button** is `disabled` until all required fields
  (`lastName` for STUDENTS) are mapped (`ImportColumnMapper.tsx:231-233`).

### Untis branch (ImportUntisPreview)

- Shows parsed teacher/class/room/lesson counts (from `parseResult`).
- **`"Weiter"` button** at `ImportUntisPreview.tsx:181` — always enabled once
  parseResult renders.

## Step 3 — ImportDryRunPreview

- Rendered when `dryRunJob?.dryRunResult` is truthy (`ImportWizard.tsx:264`).
- Two buttons (`ImportDryRunPreview.tsx:112-115`):
  - `"Zurueck"` (secondary — back to Step 2)
  - `"Daten importieren"` (primary — triggers `handleCommit` →
    `commitMutation.mutate`)

## Step 4 — ImportProgressPanel

- Socket-driven via `useImportSocket` (`ImportWizard.tsx:86-92`).
- On `import:complete` socket event, wizard auto-advances to Step 5.
- For small fixtures (≤5 rows), `handleCommit` often receives the completed
  job synchronously in the mutation response → skips Step 4 entirely
  (`ImportWizard.tsx:144-153`).

## Step 5 — ImportResultSummary

- Status → badge label (`ImportResultSummary.tsx:11-36`):
  - `COMPLETED` → `"Erfolgreich"` (green badge, CheckCircle icon)
  - `PARTIAL` → `"Teilweise"` (orange badge, AlertTriangle icon)
  - `FAILED` → `"Fehlgeschlagen"` (red badge, XCircle icon)
- `getResultText()` strings (`ImportResultSummary.tsx:58-66`):
  - `COMPLETED`: `"{imported} von {total} Datensaetzen erfolgreich importiert."`
  - `PARTIAL`: `"{imported} von {total} importiert. {skipped} uebersprungen, {errors} fehlerhaft. Details im Importbericht."`
  - `FAILED`: `"Import fehlgeschlagen. Bitte pruefen Sie die Datei und versuchen Sie es erneut."`
- Stats grid: Gesamt / Importiert / Uebersprungen / Fehler (4 columns).
- `"Neuer Import"` button resets the wizard.

## Backend validation contract (STUDENTS)

From `import.processor.ts:341-345`:

```ts
case 'STUDENTS':
  if (!row.lastName?.trim()) {
    return { status: 'error', error: 'Missing student lastName' };
  }
  return { status: 'imported' };
```

Empty `lastName` → error row → PARTIAL status when errorCount < totalRows.
Matches the 10.5-03 csv-malformed.csv design (row 2: `Anna,,anna@example.at,1A`).

## REST cleanup contract

Routes are nested under `/schools/:schoolId/import` (singular, not plural).
From `import.controller.ts:43-160`:

- `GET  /api/v1/schools/:schoolId/import` — list
- `DELETE /api/v1/schools/:schoolId/import/:importJobId` — delete (cascades FKs)

## Mobile-375 decision (D-09c)

Empirical check deferred — ImportColumnMapper has explicit mobile layout
(`sm:hidden` stacked cards, `ImportColumnMapper.tsx:101-143`), but the 5-step
wizard + dry-run preview + progress panel + result badges amplify the
WebKit `Bus error: 10` environmental gate already documented for every other
mobile-375 spec in 10.4 / 10.5-01 / 10.5-02. Mobile variant authored would run
into the same frozen WebKit blocker; deferring per CONTEXT.md D-09c
"entfällt unless Wizard is 375px-responsive" — wizard IS responsive, but the
Playwright tooling can't verify it on this host.
