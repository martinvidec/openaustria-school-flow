# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: School Data Model & DSGVO. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification: BACKEND-ONLY**

Phase 2 delivers Prisma schema entities (Teacher, Student, Class, Group, Subject), REST API endpoints, and DSGVO compliance infrastructure (consent, deletion, export, encryption, retention). There is no frontend application in the project yet -- `apps/api` is the only app, and the first UI phase is Phase 4 (Timetable Viewing). This UI-SPEC therefore focuses on API response contracts, error copywriting, and state definitions that future frontend phases will consume.

## Copywriting Contract

Phase 2 produces REST API responses that will surface in future UIs. The copywriting contract defines API-level messages, error responses (RFC 9457 Problem Detail format established in Phase 1), and state labels. All copy is in English for API responses (German localization is a UI-layer concern in later phases).

### API Error Messages (RFC 9457 Problem Detail)

These error messages are returned in the `detail` field of Problem Detail responses. They must be human-readable, actionable, and consistent with Phase 1 patterns.

| Endpoint Context | HTTP Status | Problem Detail `title` | Problem Detail `detail` |
|-----------------|-------------|----------------------|------------------------|
| Create teacher: duplicate personalNumber | 409 | Conflict | A teacher with personal number {personalNumber} already exists in this school. |
| Create teacher: invalid subject qualification | 400 | Validation Error | Subject {subjectId} does not exist. Verify the subject ID and try again. |
| Create class: duplicate name+year | 409 | Conflict | Class {name} already exists for school year {schoolYear}. |
| Create student: class at capacity | 422 | Unprocessable Entity | Class {className} has reached its maximum student capacity. |
| Assign student to group: already a member | 409 | Conflict | Student is already a member of group {groupName}. |
| Apply Stundentafel template: template not found | 404 | Not Found | No Stundentafel template found for school type {schoolType}, year level {yearLevel}. |
| DSGVO export request: export already in progress | 409 | Conflict | A data export is already in progress for this person. Please wait for it to complete. |
| DSGVO deletion: person already anonymized | 409 | Conflict | This person's data has already been anonymized. |
| DSGVO deletion: retention period active | 422 | Unprocessable Entity | Cannot delete data for category {category}. Retention period expires on {expiryDate}. |
| Consent: invalid processing purpose | 400 | Validation Error | Unknown processing purpose {purpose}. Valid purposes: Stundenplanerstellung, Kommunikation, Notenverarbeitung, Fotofreigabe, Kontaktdaten-Weitergabe, Lernplattform, Statistik. |
| Consent: already granted | 409 | Conflict | Consent for {purpose} has already been granted by this person. Use PUT to update. |
| Encryption: key not configured | 500 | Internal Server Error | Encryption service is not configured. Contact your system administrator. |
| Retention policy: duplicate category | 409 | Conflict | A retention policy for category {category} already exists for this school. Use PUT to update. |
| Generic: entity not found | 404 | Not Found | {EntityType} with ID {id} was not found. |
| Generic: unauthorized for school | 403 | Forbidden | You do not have permission to access data for this school. |

### DSGVO Operation Status Labels

BullMQ jobs return status values. These labels define the contract for job state reporting.

| Operation | Status | Label | Description |
|-----------|--------|-------|-------------|
| Data Export (Art. 15/20) | QUEUED | Export queued | Your data export request has been received and is waiting to be processed. |
| Data Export (Art. 15/20) | PROCESSING | Export in progress | Your data is being collected and packaged. This may take several minutes. |
| Data Export (Art. 15/20) | COMPLETED | Export ready | Your data export is ready for download. It will be available for 7 days. |
| Data Export (Art. 15/20) | FAILED | Export failed | An error occurred while generating your data export. Please try again or contact your administrator. |
| Data Deletion (Art. 17) | QUEUED | Deletion queued | The data deletion request has been received and is waiting to be processed. |
| Data Deletion (Art. 17) | PROCESSING | Deletion in progress | Personal data is being anonymized across all records. |
| Data Deletion (Art. 17) | COMPLETED | Deletion complete | All personal data has been anonymized. Structural records have been retained with anonymized references. |
| Data Deletion (Art. 17) | FAILED | Deletion failed | An error occurred during data anonymization. No data was modified. Please try again or contact your administrator. |
| Retention Cleanup | RUNNING | Retention check running | Daily retention check is evaluating expired records. |
| Retention Cleanup | COMPLETED | Retention check complete | {count} expired records processed. |

### Confirmation Copy for Destructive Actions

These will surface in future admin UIs but are defined here as the API contract.

| Action | Confirmation Prompt | Consequence Description |
|--------|-------------------|----------------------|
| Delete person data (Art. 17) | Permanently anonymize all personal data for {firstName} {lastName}? | All personal information (name, contact details, health data) will be replaced with anonymous placeholders. Grades, attendance records, and class history will be retained with anonymized references. This action cannot be undone. |
| Withdraw consent | Withdraw consent for {purposeLabel}? | Data processing for this purpose will stop. Existing data may be retained if a legal retention obligation applies. |
| Override retention policy | Change retention period for {category} from {currentDays} days to {newDays} days? | This will affect when expired records in this category are automatically cleaned up. Shortening the period may trigger deletion of records that were previously within retention. |
| Delete class | Remove class {className} and all student assignments? | All students will be unassigned from this class and its groups. Student records themselves will not be deleted. Stundentafel entries for this class will be removed. |

| Entity | Empty State Heading | Empty State Body |

| DSFA entries | No DSFA entries | Add data protection impact assessment entries for each processing activity in your school. |

## API Response State Contract

Phase 2 APIs return data that future UIs will render. This section defines the states each major entity can be in, so that future UI phases know what visual states to design for.

### Teacher States

| State | Condition | Visual Indicator (future UI) |
|-------|-----------|------------------------------|
| Active | Default state, not anonymized | Normal display |
| Shared | `isShared = true` | Badge or icon indicating shared/Wanderlehrer |
| Part-time | `employmentPercentage < 100` | Display percentage, reduced availability indicator |
| Anonymized | `person.isAnonymized = true` | Greyed out, "Anonymized" label, no personal data visible |

### Student States

| State | Condition | Visual Indicator (future UI) |
|-------|-----------|------------------------------|
| Active | Assigned to a Stammklasse | Normal display with class badge |
| Unassigned | No Stammklasse assignment | Warning indicator, prompt to assign |
| Anonymized | `person.isAnonymized = true` | Greyed out, "Anonymized" label |

### Class States

| State | Condition | Visual Indicator (future UI) |
|-------|-----------|------------------------------|
| Empty | No students assigned | Empty state with add prompt |
| Populated | Has students, no Stundentafel | Warning: missing Stundentafel |
| Ready | Has students AND Stundentafel applied | Green/ready indicator |

### Consent Record States

| State | Condition | Visual Indicator (future UI) |
|-------|-----------|------------------------------|
| Granted | `granted = true`, `withdrawnAt = null` | Green checkmark |
| Withdrawn | `withdrawnAt != null` | Red/crossed indicator with withdrawal date |
| Not recorded | No ConsentRecord exists for purpose | Grey/empty state |
| Retained by law | `granted = false` but `legalBasis = 'legal_obligation'` | Yellow/info: retained due to legal obligation |

### DSGVO Job States

| State | Condition | Visual Indicator (future UI) |
|-------|-----------|------------------------------|
| Queued | BullMQ job waiting | Spinner with "Queued" label |
| Processing | BullMQ job active | Progress indicator with status label |
| Completed | BullMQ job completed | Success checkmark with download link (export) or completion message (deletion) |
| Failed | BullMQ job failed | Error indicator with retry action |
