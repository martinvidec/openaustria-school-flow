Implement DSGVO data deletion (anonymization per Art. 17) and data export (Art. 15/20) with async BullMQ processing and PDF generation.

Purpose: DSGVO-02 requires right-to-be-forgotten via anonymization. DSGVO-03 requires data export including both JSON and PDF formats. DSGVO-04 is satisfied by Plan 01's encryption service -- this plan ensures encrypted fields are properly handled during export/deletion. Both are long-running operations processed asynchronously via BullMQ.

    - `async requestDeletion(dto: RequestDeletionDto)`:
      1. Check person exists, throw NotFoundException if not
      2. Check not already anonymized: throw ConflictException `This person's data has already been anonymized.`
      3. Check no pending deletion (DsgvoJob with personId + DATA_DELETION + status QUEUED/PROCESSING): throw ConflictException
      4. Create DsgvoJob with jobType=DATA_DELETION, status=QUEUED
      5. Add to BullMQ queue: `deletionQueue.add('anonymize-person', { personId, dsgvoJobId })`
      6. Update DsgvoJob with bullmqJobId
      7. Return DsgvoJob
    - `async anonymizePerson(personId: string, dsgvoJobId: string)`:
      1. Update DsgvoJob status to PROCESSING
      2. Load person with all relations
      3. In a prisma.$transaction:
         a. Generate anonymous ID: sequential counter or hash of person ID
         b. Update Person: firstName='Geloeschte', lastName='Person #${counter}', email=null, phone=null, address=null, dateOfBirth=null, socialSecurityNumber=null, healthData=null, isAnonymized=true, anonymizedAt=new Date()
         c. Update ConsentRecord entries for this person: set legalBasis to 'anonymized'
         d. Clear any PII from audit entries metadata that reference this personId (check metadata JSON for name/email patterns, replace with '[anonymized]')
      4. Update DsgvoJob status to COMPLETED
      5. If error: Update DsgvoJob status to FAILED, errorMessage = error.message
    - `async getStatus(dsgvoJobId: string)`:
      - Return DsgvoJob by id
    - `async getDeletionsByPerson(personId: string)`:
      - Return all DsgvoJobs for person with jobType=DATA_DELETION

    **DeletionProcessor** at `apps/api/src/modules/dsgvo/processors/deletion.processor.ts`:
    ```typescript
    @Processor(DSGVO_DELETION_QUEUE)
    export class DeletionProcessor extends WorkerHost {
      constructor(private dataDeletionService: DataDeletionService) { super(); }

      async process(job: Job<{ personId: string; dsgvoJobId: string }>) {
        const { personId, dsgvoJobId } = job.data;
        await this.dataDeletionService.anonymizePerson(personId, dsgvoJobId);
        return { anonymized: true, personId };
      }
    }

           "auditLog": [...]
         }
         ```
      4. Generate PDF summary via PdfExportService
      5. Store both in DsgvoJob.resultData as JSON: { jsonExport: {...}, pdfBase64: "..." }
      6. Update DsgvoJob status=COMPLETED
