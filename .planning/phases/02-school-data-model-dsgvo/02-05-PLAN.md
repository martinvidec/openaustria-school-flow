    **ConsentService:**
    - `async grant(dto: CreateConsentDto)`:
      - Check if consent already exists for person+purpose
      - If exists and granted=true: throw ConflictException `Consent for ${purpose} has already been granted by this person. Use PUT to update.`
      - If exists and withdrawn: update to re-grant (set granted=true, grantedAt=now, withdrawnAt=null, version++)
      - If not exists: create ConsentRecord with granted=true, grantedAt=now
    - `async withdraw(dto: WithdrawConsentDto)`:
      - Find consent for person+purpose, throw NotFoundException if not found
      - Set withdrawnAt=now, granted=false
      - Do NOT delete the record (withdrawal is tracked)
    - `async findByPerson(personId: string)`:
      - Return all ConsentRecord for person, ordered by purpose
    - `async findBySchool(schoolId: string, pagination: PaginationQueryDto)`:
      - Join through person.schoolId, paginated
    - `async hasConsent(personId: string, purpose: string): boolean`:
      - Returns true if consent exists, granted=true, withdrawnAt=null

    **ConsentController:** @Controller('dsgvo/consent')
    - POST / -- grant consent
    - POST /withdraw -- withdraw consent
    - GET /person/:personId -- consents for person
    - GET /school/:schoolId -- all consents for school, paginated
    - All with @CheckPermissions({ action: X, subject: 'consent' })

    - `async create(dto: CreateRetentionPolicyDto)`:
      - Check unique [schoolId, dataCategory]: throw ConflictException `A retention policy for category ${category} already exists for this school. Use PUT to update.`
      - Create with isDefault=false (admin override)
    - `async findBySchool(schoolId: string)`:
      - Return all policies for school
    - `async update(id: string, retentionDays: number)`:
      - Update retentionDays, set isDefault=false
    - `async remove(id: string)`:
      - Delete policy (reverts to system default)
    - `async getEffectivePolicy(schoolId: string, dataCategory: string)`:
      - Find school-specific policy, or return default
      - Default retention periods:
        - noten: 21900 (60 years -- Austrian Aufbewahrungspflicht)
        - anwesenheit: 1825 (5 years)
        - kommunikation: 365 (1 year)
        - audit_mutation: 1095 (3 years -- from Phase 1 D-07)
        - audit_sensitive_read: 365 (1 year -- from Phase 1 D-07)
        - personal_data: 1825 (5 years after departure)
        - health_data: 365 (1 year after purpose fulfilled)
    - `async checkExpiredRecords(schoolId: string)`:
      - For each category, find records older than retentionDays
      - Return list of { category, count, oldestDate } for reporting
      - Does NOT delete -- deletion is handled by deletion processor

    **RetentionProcessor** at `apps/api/src/modules/dsgvo/processors/retention.processor.ts`:
    ```typescript
    import { Processor, WorkerHost } from '@nestjs/bullmq';
    import { Job } from 'bullmq';
    import { DSGVO_RETENTION_QUEUE } from '../../../config/queue/queue.constants';

    @Processor(DSGVO_RETENTION_QUEUE)
    export class RetentionProcessor extends WorkerHost {
      constructor(private retentionService: RetentionService) { super(); }

      async process(job: Job<{ schoolId?: string }>) {
        // If schoolId provided, check that school only
        // Otherwise, check all schools
        const results = await this.retentionService.checkExpiredRecords(job.data.schoolId);
        return { processed: true, results };
      }
    }
    ```
