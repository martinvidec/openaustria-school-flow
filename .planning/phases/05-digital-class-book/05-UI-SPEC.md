## Spacing Scale

Declared values (inherited from Phase 4 UI-SPEC, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, attendance status icon margin |
| sm | 8px | Compact element spacing, grade matrix cell padding, attendance row line spacing |
| md | 16px | Default element spacing, tab content padding, card padding, form field gap |
| lg | 24px | Section padding, dialog padding, lesson detail page section gaps |
| xl | 32px | Layout gaps between major sections (tabs container and content, sidebar and main) |
| 2xl | 48px | Major section breaks, page-level vertical padding |
| 3xl | 64px | Page-level spacing, content max-width gutters |

Exceptions:
- Touch targets on mobile: minimum 44x44px (WCAG 2.5.5 AAA). Applies to attendance status tap targets, tab triggers, grade entry buttons, excuse action buttons (D-01, BOOK-07).
- Attendance grid row height: 48px per student row (accommodates name + status icon at 44px touch target + 4px padding).
- Grade matrix cell size: 56px width minimum per grade column (accommodates grade display "2+" at 14px + padding).

## Interaction Contracts

### Lesson Detail Page with Tabs (BOOK-01, BOOK-02, BOOK-03, BOOK-04, D-02)

| Interaction | Behavior |
|-------------|----------|
| Entry point | Teacher clicks a lesson cell in their timetable view. Navigates to `/classbook/{lessonId}` (D-03) |
| Page header | Shows subject name, class name, teacher name, day of week, period number, calendar date. Back arrow returns to timetable |
| Tab layout | 4 tabs: Anwesenheit, Inhalt, Noten, Notizen. Default tab: Anwesenheit. Tab state persisted in URL search param `?tab=anwesenheit` |
| Tab switching | Click tab trigger. Content area swaps. No page reload. Each tab fetches data independently via TanStack Query |
| Mobile tabs | Tabs scroll horizontally if they exceed viewport width. Active tab scrolls into view |
| Auto-save | Lesson content (Inhalt tab) auto-saves on field blur with 1s debounce. Toast: "Gespeichert" |
| Create-on-navigate | If no ClassBookEntry exists for this lesson+date, the frontend creates it on first save (any tab) |

### Attendance Quick-Tap Grid (BOOK-01, D-01, D-04)

| Interaction | Behavior |
|-------------|----------|
| Initial state | All students listed alphabetically (surname, first name). Each row shows: student name, status icon. Default status: anwesend (green check) |
| Tap-to-cycle | Tap student row cycles status: anwesend -> abwesend -> verspaetet -> entschuldigt -> anwesend. Icon and background color update instantly (optimistic) |
| Late minutes | When status cycles to "verspaetet", an inline number input appears to the right. Default: empty. Teacher enters minutes (e.g., "12"). Input is 48px wide |
| "Alle anwesend" button | Secondary button at top of grid. Sets all students to "anwesend". Resets any late minutes. Confirmation not required (common case, easily undone) |
| Batch save | Changes debounced 2s after last interaction. Bulk POST to backend. Loading spinner on save button during save. Toast: "Anwesenheit gespeichert" |
| Real-time sync | Socket.IO event `classbook:attendance-updated` triggers TanStack Query invalidation. Other teachers viewing same lesson see changes |
| Conflict handling | If another teacher edits the same record, server returns 409. Frontend shows toast: "Anwesenheit wurde von {teacherName} geaendert. Seite wird aktualisiert." then refetches |

### Grade Entry and Matrix (BOOK-03, D-05, D-06, D-07)

| Interaction | Behavior |
|-------------|----------|
| Grade matrix view | Spreadsheet layout: rows = students (alphabetical), columns = grade entries (chronological). Last column: weighted average (calculated, not editable) |
| Column header | Shows date + category icon (pen for Schularbeit, speech bubble for Muendlich, hand for Mitarbeit). Click to sort by that grade column |
| Grade cell display | Shows grade notation ("2+", "3", "1-"). Background tinted per grade color scale. Click opens edit dialog |
| Add grade button | "Note hinzufuegen" button above matrix. Opens GradeEntryDialog. Fields: student (select or all), category (select), value (GradeValuePicker), description (optional text), date (defaults to today) |
| Grade value picker | Compact picker showing Austrian grades. Tap "2" then tap "+" to get "2+". Or direct entry. Valid values only (0.75 to 5.25 in defined steps) |
| Grade weights | Collapsible section above matrix: "Gewichtung: SA 40% / M 30% / MA 30%". Teacher can click "Anpassen" to override per-subject. Changes recalculate all averages |
| Filter by category | Badge-style filter pills above matrix: "Alle", "Schularbeit", "Muendlich", "Mitarbeit". Active filter highlighted with accent color |
| Sort | Click column header to sort. Click student name header to sort alphabetically. Click average header to sort by average |
| Mobile view | Matrix scrolls horizontally inside ScrollArea. Student name column is sticky (position: sticky, left: 0). Average column is sticky right |
| Role-based access | Teacher: full edit. Schulleitung: read-only. Parent: sees only their child's row + weighted average. Student: sees only own row (D-08) |

### Lesson Content Documentation (BOOK-02, D-09)

| Interaction | Behavior |
|-------------|----------|
| Form fields | Three labeled textareas: "Thema" (single line, 60 char max displayed), "Lehrstoff" (multiline, expandable), "Hausaufgabe" (multiline, expandable) |
| Auto-save | Each field saves on blur with 1s debounce. No explicit save button needed. Small "Gespeichert" indicator next to field after successful save |
| Character counter | No hard limit, but Thema shows character count after 50 chars as soft guidance |
| Previous entries | Collapsible "Letzte Eintraege" section below form showing the last 3 entries for this class+subject (for reference). Read-only |

### Student Notes (BOOK-04, D-10)

| Interaction | Behavior |
|-------------|----------|
| Note list | Notes grouped by student. Each note shows: author name, timestamp, content preview (2 lines), private badge if applicable |
| Add note | "Notiz hinzufuegen" button. Inline form expands: student selector, content textarea, "Privat" checkbox. Save button |
| Private flag | Checkbox "Nur fuer mich und Schulleitung sichtbar". When checked, note shows lock icon. Visible only to author + Schulleitung (D-10) |
| Edit/delete | Dropdown menu on own notes: "Bearbeiten", "Loeschen". Delete requires confirmation dialog |
| Visibility | Teachers of the class + Schulleitung + Klassenvorstand see all non-private notes. Private notes visible only to author + Schulleitung |

### Parent Excuse Workflow (BOOK-06, D-11, D-12, D-13)

| Interaction | Behavior |
|-------------|----------|
| Entry point (parent) | Sidebar nav item "Entschuldigungen" visible to Eltern role. Also accessible via "Kind abwesend melden" button in parent dashboard |
| Excuse form | Fields: child selector (dropdown, pre-selected if single child), Von (date picker), Bis (date picker), Grund (select: Krank, Arzttermin, Familiaer, Sonstig), Anmerkung (optional textarea), Datei (optional file upload). Submit button: "Entschuldigung einreichen" |
| File upload | Drag-and-drop zone or click to browse. Accepts PDF, JPG, PNG. Max 5MB. Shows filename + size after selection. Remove button (X) to clear |
| Date validation | "Bis" date must be >= "Von" date. Cannot submit excuse for dates more than 30 days in the past |
| Submission | POST to backend. Toast: "Entschuldigung eingereicht". Excuse appears in parent's list with "Ausstehend" badge |
| Parent list view | Table/card list of submitted excuses. Columns: Kind, Zeitraum, Grund, Status (badge). Sortable by date. Filterable by status |
| Klassenvorstand review | Teachers with Klassenvorstand role see "Entschuldigungen" in sidebar. Page shows pending excuses list for their class(es) |
| Review actions | Each pending excuse card shows: student name, date range, reason, parent note, attachment (downloadable). Two buttons: "Akzeptieren" (primary), "Ablehnen" (destructive outline) |
| Accept flow | Click "Akzeptieren". Optional review note dialog. On confirm: excuse status -> ACCEPTED, all affected attendance records updated to ENTSCHULDIGT. Toast: "Entschuldigung akzeptiert" |
| Reject flow | Click "Ablehnen". Required review note dialog (reason for rejection). On confirm: excuse status -> REJECTED. Toast: "Entschuldigung abgelehnt". Parent sees rejection with note |
| Real-time | Socket.IO event `classbook:excuse-updated` notifies parent of status change. Badge count on sidebar "Entschuldigungen" shows pending count for Klassenvorstand |

### Absence Statistics (BOOK-05)

| Interaction | Behavior |
|-------------|----------|
| Entry point | "Abwesenheitsstatistik" link within classbook or dedicated sub-route. Visible to Lehrer (own classes), Schulleitung (all), Admin (all) |
| Date range filter | From/to date pickers. Defaults: current semester (September 1 to January 31 or February 1 to June 30) |
| Per-student view | Table: student name, total lessons, present count, absent (unexcused) count, absent (excused) count, late count, late count >15min, absence percentage. Sortable by any column |
| Per-class summary | Aggregated card: total lessons, average attendance rate, most absent students (top 5), trend line (not MVP -- defer chart to v2) |
| >15min lateness | Late arrivals >15min highlighted with warning color. Counted separately as they count as absence per Austrian Schulunterrichtsgesetz (D-04) |
| Export | "Als PDF exportieren" button. Generates PDF of the statistics table for the selected date range and class |

**Copywriting conventions (inherited from Phase 4):**
- Formal "Sie" address (standard in Austrian school administration)
- No exclamation marks in error messages
- Error messages always include a next-step action
- Button labels use imperative verb + noun
- Confirmation dialogs for destructive actions always state the consequence explicitly
- Toast messages are concise (under 60 characters) and disappear after 4 seconds
- Toast messages for real-time updates disappear after 3 seconds

## Accessibility Contract

Inherited from Phase 4 UI-SPEC with Phase 5 additions:

| Requirement | Implementation |
|-------------|---------------|
| WCAG 2.1 AA contrast | All attendance status colors meet 4.5:1 on their backgrounds. Grade colors use foreground text (not colored text) |
| Keyboard navigation | Tab through attendance grid rows. Enter/Space to cycle status. Arrow keys in grade matrix. Tab through form fields |
| Screen reader | Attendance row: `aria-label="{studentName}: {status}"`. Grade cell: `aria-label="Note {value} in {category} vom {date}"`. Excuse card: `aria-label="Entschuldigung fuer {studentName}, {dateRange}, Status: {status}"` |
| Focus indicators | 2px outline in accent color (`--primary`) with 2px offset on all interactive elements (inherited) |
| Touch targets | Minimum 44x44px on attendance status icons, grade cells, tab triggers, excuse action buttons (BOOK-07) |
| Live regions | Attendance save confirmation, excuse status change announcements via `aria-live="polite"` |
| Language | `<html lang="de">` (inherited) |
| Form labels | All form fields have associated `<Label>` elements. Required fields marked with `aria-required="true"` |
| Error announcements | Form validation errors announced via `aria-describedby` linked to error message elements |
