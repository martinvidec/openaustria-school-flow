# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Substitution Planning. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification: THIRD FRONTEND PHASE.** All design tokens (spacing, typography, base colors, semantic colors, subject palette, change indicator colors) are inherited verbatim from Phase 4 and Phase 5 UI-SPECs. Phase 6 adds substitution-specific components, interaction patterns, notification center UI, and copywriting within the established design system. The `stillarbeit` ChangeIndicator variant is the only new visual token.

## Spacing Scale

Declared values inherited from Phase 4 UI-SPEC (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, notification bell badge offset, score breakdown row gap |
| sm | 8px | Compact element spacing, candidate row internal padding, notification dropdown item padding |
| md | 16px | Default element spacing, card padding, absence form field gap, handover note field gap |
| lg | 24px | Section padding, dialog padding, `/admin/substitutions` panel padding |
| xl | 32px | Layout gaps between major sections (three panels on admin page, header and content) |
| 2xl | 48px | Major section breaks, page-level vertical padding |
| 3xl | 64px | Page-level spacing, content max-width gutters |

Exceptions (Phase 6 specific):
- Touch targets on mobile: minimum 44x44px (WCAG 2.5.5 AAA). Applies to Accept/Decline buttons on substitute offer view, notification bell trigger, absence-form date inputs, candidate list row tap targets (BOOK-07 principle applies to Phase 6, per D-22).
- Notification dropdown width: 320px (desktop), full viewport width minus 16px gutter on mobile.
- Notification list max-height: 480px with internal scroll; individual notification row height 72px (2 lines of body + timestamp + 12px padding).
- Candidate list row height: 64px (name line + score breakdown line + 12px padding).
- Fairness stats table row height: 48px (compact data density, horizontally scrollable on mobile).

**Accent reserved for (exhaustive list for Phase 6):**

2. Active tab indicator on `/admin/substitutions` (Abwesenheiten / Offene Vertretungen / Statistik)

4. Notification bell "has unread" state ring when dropdown is open

7. "Neu" (new) badge on unread notification rows

**Not permitted for:** generic hover states (use `--color-muted`), generic borders (use `--color-border`), decorative backgrounds, Accept button success state (use `--color-success` semantic token instead).

### Change Indicator Colors (inherited from Phase 4 UI-SPEC, extended by Phase 6)

Phase 4 established three indicator variants. Phase 6 adds a fourth (`stillarbeit`) without inventing new colors — it reuses the substitution/change orange scheme plus a distinct label token.

| Indicator | Border Color | Background | Phase 6 Usage |
|-----------|-------------|------------|---------------|
| Substitution/Change (`substitution`) | hsl(25 95% 53%) / orange-500 | hsl(33 100% 96%) / orange-50 | SUBSTITUTED overlay: strikethrough original teacher, new teacher below. Inherited from Phase 4. |
| Cancelled (`cancelled`) | hsl(0 84% 60%) / red-500 | hsl(0 86% 97%) / red-50 | ENTFALL overlay: "Entfall" badge, strikethrough all content. Inherited from Phase 4. |
| Room change (`room-change`) | hsl(221 83% 53%) / blue-500 | hsl(214 95% 93%) / blue-50 | Substitute with different room: strikethrough old room, arrow, new room. Inherited from Phase 4. May combine with `substitution` variant. |
| **Stillarbeit (`stillarbeit`, NEW Phase 6)** | hsl(25 95% 53%) / orange-500 | hsl(33 100% 96%) / orange-50 | Orange border (same as substitution) + "Stillarbeit" text replaces subject abbreviation + supervising teacher name below. Reuses substitution visual scheme to signal "supervised instead of normal lesson" while being lexically distinct via the label. |

**Rationale for Stillarbeit reuse of substitution colors:** Stillarbeit is a class of substitution (D-04: "Three first-class lesson outcomes per affected lesson: SUBSTITUTED, ENTFALL, STILLARBEIT"). Using a fourth unique color would violate the 60/30/10 contract. The differentiation happens via text label ("Stillarbeit" replaces subject abbrev) and a small "SA" icon badge, not via new color.

### Fairness Delta Visual Encoding

| Delta range (per D-17) | Color | Interpretation |
|------------------------|-------|----------------|
| `delta <= 0` (at or below school avg) | `--color-success` | Fair load |
| `0 < delta <= +2` | `--color-muted-foreground` | Slight overload (neutral, informational) |
| `+2 < delta <= +5` | `--color-warning` | Notable overload |
| `delta > +5` | `--color-destructive` | Severe overload (fairness concern) |

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|

### Absence Recording Flow (SUBST-01, D-01, D-02, D-03)

| Interaction | Behavior |
|-------------|----------|
| Open form | Click "Neue Abwesenheit erfassen" in Tab 1. Card expands, focus moves to Lehrer select. |
| Teacher select | Single-select dropdown, searchable, shows "{Surname}, {FirstName}" of teachers at the admin's school. |
| Date range | Two native `<input type="date">` fields: "Von" and "Bis". Validation: Bis >= Von. Default Von = today, default Bis = today. |
| Period range (optional) | Two `<input type="number">` fields: "Von Stunde" and "Bis Stunde". Both optional. If present, range must satisfy 1..12 and Bis Stunde >= Von Stunde. If both empty, absence applies to the full day. |
| Reason select | Required. Fixed taxonomy: "Krank", "Fortbildung", "Dienstreise", "Schulveranstaltung", "Arzttermin", "Sonstiges" (enum values `KRANK`/`FORTBILDUNG`/`DIENSTREISE`/`SCHULVERANSTALTUNG`/`ARZTTERMIN`/`SONSTIGES`). |
| Note | Optional Textarea, placeholder "Zusaetzliche Informationen (optional)". |
| Submit CTA | "Abwesenheit erfassen" (primary). POST /absences. Server runs range expansion (D-02) and creates pending Substitution rows per affected lesson. |
| Success | Toast: "Abwesenheit erfasst. {N} Stunden betroffen." Tab 1 form collapses and resets. Tab badge on "Offene Vertretungen" updates with new pending count. |
| Error: no lessons affected | Toast warning: "Keine Unterrichtsstunden im angegebenen Zeitraum gefunden." Form stays open. |
| Error: generic | Inline field errors for validation. Sonner error toast for server errors with retry path. |

### Substitute Assignment Flow (SUBST-02, D-05, D-06, D-08)

| Interaction | Behavior |
|-------------|----------|
| Open row | Admin clicks a pending Substitution row in Tab 2. Row expands to reveal `AssignmentActions` + `CandidateList` (collapsed). |
| Show candidates | Admin clicks "Vertretung anbieten" button. `CandidateList` expands below row. Backend fetches ranked candidates synchronously (NestJS service, not BullMQ). Loading spinner shown until data arrives. |
| Candidate row | Rank number, teacher name, score as percentage ("87%"), `ScoreBreakdownRow` showing 4 factor values, Klassenvorstand badge if applicable, "Anbieten" button (secondary). |
| Top rank highlight | Row #1 has 1px solid accent left border + slight primary-tinted background to draw the eye. |
| Offer action | Click "Anbieten" on a candidate row. Confirmation dialog: "Vertretung an {Name} anbieten?" with candidate score breakdown summary. Confirm button "Anbieten". |
| On confirm | PATCH substitution status to OFFERED, create SUBSTITUTION_OFFER notification, emit on `/notifications` socket. Toast: "Vertretung angeboten an {Name}". Substitution row status badge updates to "Angeboten". |
| Stale candidate (Pitfall 2) | If the chosen candidate is no longer free at the time of confirm (409 Conflict), toast error: "Vertretung kann nicht vergeben werden: Lehrer/in ist nicht mehr verfuegbar. Liste wird aktualisiert." Auto-refresh candidate list. |
| Entfall action | Click "Entfall" button on `AssignmentActions`. Confirmation dialog: "Stunde als Entfall markieren?" with warning that no ClassBookEntry will be created. Confirm button "Als Entfall markieren" (destructive outline). |
| On entfall confirm | PATCH substitution to type=ENTFALL, status=CONFIRMED. Emit `timetable:cancelled` on `/timetable` socket. Toast: "Stunde als Entfall markiert". |
| Stillarbeit action | Click "Stillarbeit" button on `AssignmentActions`. Dialog: optional supervising teacher selector (can be left empty — any available teacher supervises per D-04). Confirm button "Als Stillarbeit markieren". |
| On stillarbeit confirm | PATCH substitution to type=STILLARBEIT, status=CONFIRMED. Create ClassBookEntry with thema="Stillarbeit" marker (per Open Question 4 recommendation, D-14). Emit `timetable:substitution` with `stillarbeit` changeType payload. Toast: "Stillarbeit eingerichtet". |
| Reassign after decline | When status=DECLINED, "Offene Vertretungen" panel surfaces the row with a "Erneut zuweisen" CTA. Clicking it reopens the candidate list. Admin can pick a different candidate. (D-06, D-07) |

- Button labels use imperative verb + noun.

- Real-time update toasts disappear after 3 seconds.
- Umlauts are ASCII-transliterated in copy (ue/oe/ae/ss) to match Phase 4/5 convention — the frontend renders them correctly via Unicode where needed but source strings stay ASCII-safe for diff stability.

## Accessibility Contract

Inherited from Phase 4/5 UI-SPEC with Phase 6 additions:

| Requirement | Implementation |
|-------------|---------------|
| WCAG 2.1 AA contrast | All new color assignments meet 4.5:1 on their backgrounds. Fairness delta color encoding verified against background pairs. Unread notification badge primary-on-white meets 7.1:1. |

Substitution status badge: `aria-label="Status: {statusText}"`. Fairness delta: `aria-label="{deltaText}"`. |
