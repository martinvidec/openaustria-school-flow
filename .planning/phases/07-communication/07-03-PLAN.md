1. **File attachment handling in MessageService** (COMM-04, D-14):

   Add methods to MessageService:

   - `async uploadAttachment(schoolId: string, messageId: string, userId: string, file: MultipartFile): Promise<MessageAttachmentDto>`:
     * Verify user is the sender of the message.
     * Validate file size <= 5MB (5 * 1024 * 1024 bytes). Throw BadRequestException if exceeded.
     * Validate MIME type against whitelist: `['application/pdf', 'image/jpeg', 'image/png', 'application/msword', 'application/vnd.openxmlformats-officedocument.wordprocessingml.document']`.
     * Magic byte validation: Read first 4 bytes of file buffer. PDF: starts with `[0x25, 0x50, 0x44, 0x46]` (%PDF). JPG: starts with `[0xFF, 0xD8, 0xFF]`. PNG: starts with `[0x89, 0x50, 0x4E, 0x47]`. DOC/DOCX: starts with `[0xD0, 0xCF, 0x11, 0xE0]` (OLE) or `[0x50, 0x4B, 0x03, 0x04]` (ZIP/DOCX).
     * Sanitize filename: replace non-alphanumeric chars (except . and -) with _.
     * Storage path: `uploads/${schoolId}/messages/${messageId}/${sanitizedFilename}`.
     * Write file to disk using `fs.promises.mkdir` (recursive) + `fs.promises.writeFile`.
     * Create MessageAttachment Prisma row with filename, storagePath, mimeType, sizeBytes.
     * Return MessageAttachmentDto.

   - `async downloadAttachment(attachmentId: string, userId: string): Promise<{ path: string, filename: string, mimeType: string }>`:
     * Fetch MessageAttachment with message -> conversation -> conversationMembers.
     * Verify user is a ConversationMember. Throw ForbiddenException if not.
     * Return storagePath, filename, mimeType for the controller to stream.

   Update MessageController:
   - `POST /schools/:schoolId/conversations/:conversationId/messages/:messageId/attachments` -- Multipart upload using `@Req() request: FastifyRequest` and `request.file()` per @fastify/multipart pattern.
   - `GET /schools/:schoolId/conversations/:conversationId/messages/:messageId/attachments/:attachmentId/download` -- Stream file using `reply.sendFile()` or pipe the read stream. Set Content-Disposition header.

1. **PollService** (apps/api/src/modules/communication/poll/poll.service.ts):

   Injectable service with PrismaService injection. Methods:

   - `async createWithMessage(conversationId: string, senderId: string, messageBody: string, pollDto: CreatePollDto): Promise<MessageDto>` -- Per D-09, D-12:
     * Within a prisma.$transaction:
       - Create Message row with type=POLL, body=messageBody, senderId.
       - Create Poll row with messageId, question=pollDto.question, type=pollDto.type, deadline=pollDto.deadline || null, isClosed=false.
       - Create PollOption rows for each option text in pollDto.options with order=index.
       - Create MessageRecipient rows for all conversation members (same as MessageService.send logic). Increment ConversationMember.unreadCount for recipients.
     * Create MESSAGE_RECEIVED Notification for recipients.
     * Return MessageDto with embedded PollDto.

   - `async castVote(pollId: string, userId: string, dto: CastVoteDto): Promise<PollDto>` -- Per D-09, D-11:
     * Fetch poll with options. Check isClosed -- throw BadRequestException("Umfrage ist geschlossen") if closed. Check deadline -- if deadline < now, auto-close (set isClosed=true) and throw BadRequestException("Frist abgelaufen").
     * Verify user is a ConversationMember of the poll's message's conversation.
     * For SINGLE_CHOICE: dto.optionIds must have exactly 1 entry. Delete any existing PollVote for this user on any option of this poll (vote change), then create new PollVote.
     * For MULTIPLE_CHOICE: dto.optionIds can have 1+. Delete all existing PollVotes for this user on this poll, then createMany new PollVotes for each optionId.
     * Return updated PollDto with recalculated counts.

   - `async closePoll(pollId: string, userId: string): Promise<PollDto>` -- Per D-11:
     * Verify user is the sender of the poll's message OR admin/schulleitung.
     * Set isClosed=true on Poll row.
     * Return PollDto.

   - `async getResults(pollId: string, userId: string): Promise<PollDto>` -- Per D-10:
     * Fetch poll with options and votes (including voter person data).
     * Determine if user is the message sender or admin/schulleitung.
     * If sender/admin: return PollOptionDto with voters array populated (named results).
     * If other user: return PollOptionDto with voters=undefined (anonymous counts only).
     * Set userVoteOptionIds to the optionIds the requesting user voted for.
