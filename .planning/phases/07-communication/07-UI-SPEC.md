# Phase 7 — UI Design Contract

> Visual and interaction contract for Phase 7: Communication. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase classification: FOURTH FRONTEND PHASE.** All design tokens (spacing, typography, base colors, semantic colors, subject palette, change indicator colors) are inherited verbatim from Phase 4, 5, and 6 UI-SPECs. Phase 7 adds messaging-specific components, a list-detail split-view layout, inline poll rendering, read receipt indicators, and copywriting within the established design system. No new color tokens are introduced.

## Spacing Scale

Declared values inherited from Phase 4 UI-SPEC (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, read receipt checkmark spacing, unread dot offset |
| sm | 8px | Compact element spacing, message bubble internal padding between meta and body, conversation list item icon-to-text gap |
| md | 16px | Default element spacing, message bubble padding, compose form field gap, conversation list item padding, poll option gap |
| lg | 24px | Section padding, compose dialog padding, conversation header padding, poll card padding |
| xl | 32px | Layout gaps between conversation list and detail panel, page-level header to content gap |
| 2xl | 48px | Major section breaks, page-level vertical padding |
| 3xl | 64px | Page-level spacing, content max-width gutters |

Exceptions (Phase 7 specific):
- Touch targets on mobile: minimum 44x44px (WCAG 2.5.5 AAA). Applies to conversation list rows, send button, poll vote buttons, attachment remove button, compose dialog action buttons, "Abwesenheit melden" quick-action button.
- Conversation list item height: 72px (avatar placeholder + 2 lines of preview + timestamp, with 14px vertical padding).
- Message bubble max-width: 70% of detail panel width (desktop/tablet), 85% on mobile. Prevents wall-of-text readability issues.
- List-detail split: conversation list panel width 360px on desktop (fixed), detail panel fills remaining width. On mobile, list and detail are separate views (not side-by-side).
- Poll option row height: 48px per option (accommodates option text + vote button at 44px touch target).

## Interaction Contracts

### `/messages` List-Detail Layout (D-05, D-06)

| Interaction | Behavior |
|-------------|----------|
| Entry point | Sidebar nav "Nachrichten" for all authenticated roles. Route `/messages`. |
| Page layout | Display title "Nachrichten" (28px) above the split view. Desktop: fixed 360px conversation list on left, flexible detail panel on right. Mobile: show list OR detail, not both. |
| Conversation list | `ConversationList` component. Sorted by last message timestamp descending. Search bar at top filters by conversation title or participant name. "Neue Nachricht" button below search opens `ComposeDialog`. |
| Select conversation | Click a row in conversation list. Detail panel loads the conversation. On mobile: navigates to `/messages/{conversationId}` (separate view). Conversation marked as read on open (PATCH all unread MessageRecipient rows for current user). |
| Default state (desktop) | When no conversation is selected: detail panel shows empty state with envelope icon and "Waehlen Sie eine Unterhaltung aus oder starten Sie eine neue Nachricht." |
| Back navigation (mobile) | Back arrow in conversation header returns to list view. Uses browser history (back button works). |
| URL state | Desktop: `/messages?id={conversationId}` preserves selected conversation on refresh. Mobile: `/messages/{conversationId}` as separate route. |
| Access control | All authenticated roles can access /messages. Message visibility is scoped by RBAC: parents see only conversations they are a member of, teachers see class conversations for their classes + direct messages, admin/schulleitung see all school conversations. |

| Interaction | Behavior |
|-------------|----------|

### Real-Time Events (Phase 7 additions)

| Event | Namespace | Client Behavior |

## Accessibility Contract

Inherited from Phase 4/5/6 UI-SPEC with Phase 7 additions:

| Requirement | Implementation |
|-------------|---------------|
| WCAG 2.1 AA contrast | All message bubble colors (own: primary/0.08 background, other: card background) meet 4.5:1 with foreground text. Read receipt icon colors meet 4.5:1 on white. Poll result bars meet contrast requirements via foreground text (not bar color). |
| Keyboard navigation | Tab through conversation list items (each item focusable, Enter selects). Tab to reply input. Tab through compose dialog fields. Enter sends message (Shift+Enter for newline). Escape closes dialogs and compose. Arrow keys navigate conversation list. |
| Screen reader | Conversation list item: `aria-label="{title}, {lastMessage preview}, {timestamp}, {unreadCount} ungelesen"`. Message bubble: `aria-label="{senderName}: {messageBody}, {timestamp}"`. Read receipt: `aria-label="{status}: {count}"`. Poll option: `aria-label="Option: {text}, {voteCount} Stimmen, {percent}%"`. Scope badge: `aria-label="Empfaengerkreis: {scope}"`. |
| Focus indicators | 2px outline in accent color (`--color-primary`) with 2px offset on all interactive elements. Conversation list items, message actions, compose fields, send button, poll options all have visible focus ring. |
| Touch targets | Minimum 44x44px on conversation list items (72px row height exceeds this), send button, attachment button, poll vote buttons/options, compose dialog buttons, absence quick-action button. |
| Live regions | New message arrival in conversation: `aria-live="polite"` on the message list container. New conversation notification via Sonner built-in live region. |
| Language | `<html lang="de">` (inherited). |
| Form labels | All compose dialog fields have associated `<Label>` elements. Required fields marked with `aria-required="true"`. Scope and type selectors use semantic native `<select>` under Radix wrapper. |
| Error announcements | Form validation errors announced via `aria-describedby` linked to error message elements. Inline field errors use `role="alert"`. |
| Motion preference | Dialog and popover animations respect `prefers-reduced-motion` (Radix default). Message list scroll animations also honor it. |
| Chat landmark | Message list uses `role="log"` with `aria-live="polite"` for assistive technology to announce new messages. |
