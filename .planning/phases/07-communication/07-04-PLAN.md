1. **MessagingGateway** (apps/api/src/modules/communication/messaging.gateway.ts):

   Create a new WebSocketGateway following the EXACT pattern of NotificationGateway. Per D-08:

   ```typescript
   @WebSocketGateway({
     namespace: 'messaging',
     cors: { origin: '*' },
     transports: ['websocket', 'polling'],
   })
   export class MessagingGateway implements OnGatewayConnection, OnGatewayDisconnect {
   ```

   Copy the JWT handshake verification logic verbatim from NotificationGateway:
   - Constructor takes ConfigService, creates JwksClient with KEYCLOAK_URL/KEYCLOAK_REALM.
   - handleConnection: extract token from handshake.auth.token or Authorization header, decode with jwt.decode, fetch signing key via jwksClient.getSigningKey(kid), verify with jwt.verify (RS256, issuer check), join `user:${payload.sub}` room.
   - handleDisconnect: log disconnection.

   Add emission methods (per 07-RESEARCH Pattern 3):

   - `emitNewMessage(recipientUserIds: string[], message: MessageDto): void` -- Emit 'message:new' to each `user:${userId}` room. Per Pitfall 3: for large broadcasts, this is O(N) per recipient. Acceptable for v1 where school-wide is ~500 max.

   - `emitReadReceipt(senderUserId: string, messageId: string, readBy: string, readCount: number, totalRecipients: number): void` -- Emit 'message:read' to `user:${senderUserId}` room only (sender sees read receipt updates).

   - `emitPollVote(recipientUserIds: string[], pollId: string, results: PollOptionDto[]): void` -- Emit 'poll:vote' to each `user:${userId}` room.

   - `emitNewConversation(memberUserIds: string[], conversation: ConversationDto): void` -- Emit 'conversation:new' to each member's room.
