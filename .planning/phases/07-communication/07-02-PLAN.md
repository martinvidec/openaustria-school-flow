1. **MessageService** (apps/api/src/modules/communication/message/message.service.ts):

   Injectable service with PrismaService, ConversationService, and NotificationService injection. Methods:

   - `async send(conversationId: string, senderId: string, dto: SendMessageDto): Promise<MessageDto>` -- Per D-02:
     * Verify sender is a ConversationMember (throw ForbiddenException if not).
     * Create Message row with type=TEXT, body=dto.body, senderId.
     * Get all member userIds via ConversationService.getMemberUserIds().
     * Create MessageRecipient rows for all members EXCEPT the sender using `prisma.messageRecipient.createMany()` with deliveredAt=new Date().
     * Increment ConversationMember.unreadCount for all recipients (not sender) using `prisma.conversationMember.updateMany({ where: { conversationId, userId: { not: senderId } }, data: { unreadCount: { increment: 1 } } })`.
     * Create MESSAGE_RECEIVED Notification for each recipient via NotificationService.create() with: type='MESSAGE_RECEIVED', title=`Neue Nachricht von ${senderName}`, body=message body truncated to 100 chars, payload={ conversationId, messageId }. Per Pitfall 2: if recipients > 50, batch notification creation rather than looping. Use createMany for MessageRecipient and loop NotificationService.create() (which handles the Socket.IO emission internally).
     * Resolve senderName from prisma.person.findFirst({ where: { keycloakUserId: senderId } }).
     * Return MessageDto with readCount=0, totalRecipients=recipientCount.

   - `async findAll(conversationId: string, userId: string, cursor?: string, limit: number = 20): Promise<{ messages: MessageDto[], nextCursor: string | null }>` -- Cursor-based pagination. Verify user is a member. Query messages ordered by createdAt DESC with take: limit+1. If result has limit+1, pop last and use its id as nextCursor. Include sender person name, attachments, poll with options. For each message where senderId === userId, include readCount (count of MessageRecipient where readAt is not null) and totalRecipients (count of MessageRecipient).

   - `async markRead(conversationId: string, userId: string): Promise<void>` -- Per COMM-03:
     * Update all MessageRecipient rows where message.conversationId = conversationId AND userId = userId AND readAt IS NULL: set readAt = now().
     * Reset ConversationMember.unreadCount to 0 for this user+conversation.
     * For each newly-read message, determine the senderId and compute the new readCount. Emit via Socket.IO (this will be wired in Plan 04 when MessagingGateway exists -- for now, just update the DB).

   - `async delete(messageId: string, userId: string): Promise<void>` -- Soft check: only sender or admin can delete. Hard delete the Message row (cascades to MessageRecipient, MessageAttachment).
