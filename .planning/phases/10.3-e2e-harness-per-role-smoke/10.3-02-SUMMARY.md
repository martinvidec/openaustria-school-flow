## Decisions Made

See frontmatter `key-decisions`. Core trade-offs:

- **Stable route-mount signal over data signal**: the `h1 "Stundenplan"`/`"Vertretungsplanung"` assertion is data-layer-independent (renders before hooks resolve) — proves auth-gate pass with zero seed-data coupling. Tier 3 hardening phases add data-dependent assertions on top.
- **Grid-or-empty-state for timetable smokes**: the seed DB ships 0 perspective-filtered lessons for kc-* personas (the solver is never invoked for them during `prisma db seed`). A grid-only assertion would false-fail on a fresh seed; an empty-only assertion would regress to green if the grid silently broke. `.or()` covers both valid render states.
- **No SMOKE-ADMIN-01**: admin coverage already lives in Phase 10.2 specs (SCHOOL-01..05, ZEIT-01..02, YEAR-02..03, WOCH-01, SILENT-4XX-01..04). Adding a 5th smoke would be pure redundancy and violate the plan's explicit must_have.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] CardTitle-is-a-div spec selector bug**

- **Found during:** Task 1 first-run verification (3/4 tests failed on `getByRole('heading', {name:'Kein Stundenplan vorhanden'})` with `element(s) not found`, SL-01 passed because `Vertretungsplanung` is a real `<h1>`)
- **Issue:** The plan draft's fallback-selector suggestion (`role="grid"` or `.timetable-grid` or `[data-testid="timetable-grid"]`) pointed only at the grid root; when verifying on the freshly-seeded dev DB the `/timetable` route rendered the empty-state card (`Kein Stundenplan vorhanden` in a shadcn `CardTitle`). I initially used `getByRole('heading', {name:...})` as the fallback for the empty state — but shadcn's `CardTitle` is a plain `<div className="text-2xl font-semibold">`, NOT a heading element. Playwright's `getByRole('heading', ...)` therefore never matched, and the `.or()` compound was effectively grid-only, which failed for all 3 non-grid personas.
- **Fix:** Switched the empty-state fallback selector from `getByRole('heading', {name:'Kein Stundenplan vorhanden'})` to `getByText('Kein Stundenplan vorhanden')`. Also tightened the h1 assertion to `getByRole('heading', {level:1, name:'Stundenplan'})` to disambiguate from the sidebar nav link (which also has accessible name "Stundenplan" but is a `link`, not a heading — the test would have passed without `level:1`, but explicit is better than implicit).
- **Files modified:** `apps/web/e2e/roles-smoke.spec.ts` (edit before commit)
- **Verification:** 4/4 green after fix (5.3s). Desktop regression 20/23 (3 pre-existing failures unchanged).
- **Committed in:** `4814dcc` (single Task 1 commit — fix landed before first commit, no separate fix-up commit)
