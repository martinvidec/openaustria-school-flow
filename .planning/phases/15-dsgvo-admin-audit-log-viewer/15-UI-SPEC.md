8-point Tailwind scale (rems βÜ£ px at default 16px root). Multiples of 4 only. Phase 15 uses Tailwind utility tokens (`p-1`, `gap-2`, etc.) βÄî not custom CSS βÄî so the scale is the Tailwind defaults below.

| Token | Value | Tailwind | Phase 15 usage |
|-------|-------|----------|---------------|
| xs | 4px | `gap-1`, `p-1` | Icon βÜî label gap inside buttons; badge inner padding |
| sm | 8px | `gap-2`, `p-2` | Filter-toolbar field-internal gaps; table cell vertical padding |
| md | 16px | `gap-4`, `p-4` | Default card padding; stack between toolbar fields; dialog body gap |
| lg | 24px | `gap-6`, `p-6` | `PageShell` content padding; section gap between filter toolbar and table |
| xl | 32px | `gap-8`, `mb-8` | Tab-panel top margin; gap between major page sections (e.g. filter + table + drawer trigger) |

## Interaction Contracts

These are not in the template's six dimensions but the orchestrator marks them as **must-lock**. Listed here so the executor and ui-checker can verify directly.

### Tab deep-linking (D-04 + D-26 precision)

- URL search-param: `?tab=consents|retention|dsfa-vvz|jobs` on `/admin/dsgvo`. For `dsfa-vvz`: `&sub=dsfa|vvz`.
- URL search-param: `?startDate&endDate&action&resource&userId&category&page` on `/admin/audit-log`.
- Implementation: `Route.useSearch()` with a route-level Zod `validateSearch` schema feeding a `useState` initializer in `DsgvoTabs.tsx` / `AuditFilterToolbar.tsx`. **No `useTab` hook** βÄî the Phase 14 reference `SolverTuningTabs.tsx` does NOT use one despite CONTEXT D-04's wording; D-26 corrects this.
- Tab change writes back to URL via `navigate({ search })` so deep-link + back-button work.

### BullMQ polling (D-13 / D-14)

- TanStack Query `refetchInterval: (query) => isTerminal(query.state.data?.status) ? false : 2000`.
- Terminal statuses: `completed`, `failed`, `cancelled`.
- Pattern source: `apps/web/src/hooks/useImport.ts:127-141`.
- Hooks created: `useDsgvoExportJob(jobId)`, `useDsgvoDeletionJob(jobId)`, `useDsgvoJobs(filters)` (list βÄî terminal-aware refetch not applicable; standard staleTime).
- Failure UX: inline warn banner below the row/dialog when `query.isError && !query.isRefetching`. NO toast on transient polling failures.

### Mutation invariants (D-20 / D-21 / C-8)

- Every `useMutation` hook has `onError: (err) => toast.error(extractMessage(err))` + `onSuccess: () => { toast.success(...); queryClient.invalidateQueries({ queryKey: [...] }); }`.
- All table rows carry `data-*` attributes for E2E selectors:
  - `<tr data-consent-id="..." data-consent-status="GRANTED|WITHDRAWN">` (Tab 1)
  - `<tr data-retention-category="...">` (Tab 2)
  - `<tr data-dsfa-id="...">` / `<tr data-vvz-id="...">` (Tab 3)
  - `<tr data-dsgvo-job-id="..." data-dsgvo-job-status="pending|running|completed|failed|cancelled">` (Tab 4)
  - `<tr data-audit-id="..." data-audit-action="create|update|delete|read">` (Audit-Log)

### Sidebar (D-22)

- Two new entries appended to admin section of `AppSidebar.tsx` with `roles: ['admin']`. Order: existing entries βÜ£ `DSGVO-Verwaltung` βÜ£ `Audit-Log`.
- Icons: `ShieldCheck` for DSGVO, `ScrollText` for Audit-Log (lucide-react).
- Active-route highlight uses existing primary-color treatment from `AppSidebar.tsx:106-108`.

### CSV export ("WYSIWYG" βÄî D-16 + D-25)

- Button placed in Audit-Log filter toolbar (variant=default, leading `Download` icon).
- Triggers `useAuditCsvExport(filters)` hook βÜ£ `apiFetch('/api/v1/audit/export.csv?' + qs)`.
- Server respects every active filter param. Browser receives `text/csv; charset=utf-8` with BOM and `Content-Disposition: attachment; filename="audit-log-{YYYY-MM-DD}.csv"`.
- Delimiter: **semicolon** per D-25 (DACH/Excel default). UTF-8 BOM mandatory so Excel auto-detects encoding.
- Hook generates a synthetic `<a download>` click then `URL.revokeObjectURL` (sketch in RESEARCH ¬ß 7).

### Audit detail drawer (AUDIT-VIEW-02)

- Triggered from row "Detail ΟΕffnen". Uses `Sheet` primitive (`apps/web/src/components/ui/sheet.tsx`) βÄî right-side slide-in, `w-[480px]` desktop / `w-full` mobile.
- Layout (top-to-bottom):
  1. `SheetHeader`: action chip + resource chip + timestamp (date-fns formatted `dd.MM.yyyy HH:mm:ss`) + actor email
  2. `Separator`
  3. Section "Vorzustand": if `entry.before` βÜ£ `<JsonTree value={entry.before} />`; else inline muted banner copy from Empty States table above.
  4. `Separator`
  5. Section "Nachzustand": `<JsonTree value={entry.metadata.body ?? entry.metadata} />`
- `JsonTree` styling: monospace 12px, `pl-4` per nesting level, color tokens βÄî string=`text-primary`, number=`text-foreground`, boolean=`text-foreground font-medium`, null=`text-muted-foreground italic`. Object/array nodes show key in `text-foreground font-medium`.

### Mobile fallback (Phase 14 carry-forward)

- Tabs render as `<ToggleGroup>` below `md` breakpoint (768px) per Phase 14 D-04 + MOBILE-ADM-01/02.
- Filter toolbar wraps via `flex-wrap`; field labels stack above inputs at `<sm` (640px).
- Audit-Log table on mobile: native `<table>` with `overflow-x-auto` wrapper. Mobile-card alternative is **DEFERRED** (Phase 16 MOBILE-ADM-01 βÄî confirmed by CONTEXT.md "Out of scope" line 24).
