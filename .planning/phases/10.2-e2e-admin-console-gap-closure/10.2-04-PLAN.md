<objective>
Lift the Plan 10.1-01 silent-4xx-toast invariant from unit-level (useMutation hook specs) to E2E level. Each of the 4 Phase-10 admin-console mutations gets one Playwright test that:
1. Mocks the network to return 4xx via `page.route()`
2. Triggers the save flow in the real UI
3. Asserts red toast appears AND green toast never appears

This catches the exact UAT bug class the user saw in Phase 10.1 — where a green success toast masked actual data loss. The 10.1-01 hook specs can be subtly wrong (mock-scope mismatch, hook-internal plumbing changes) but the E2E test exercises the real render layer + toast portal + error-handler chain.
</objective>

<action>
Create `apps/web/e2e/silent-4xx.spec.ts` with describe `Phase 10.2 — Silent-4xx toast guardrail (desktop)` and four tests. For each test use `page.route()` to intercept the specific mutation endpoint and return a 422 with a German error message, then trigger the mutation through the UI, then assert both invariants:

1. **Test `SILENT-4XX-01: Stammdaten PUT 4xx → red toast, no green toast`:**
   - `await loginAsAdmin(page)`
   - `await page.route('**/api/v1/schools/seed-school-bgbrg-musterstadt', async (route, request) => { if (request.method() === 'PUT') { await route.fulfill({ status: 422, contentType: 'application/json', body: JSON.stringify({ statusCode: 422, message: 'Validierung fehlgeschlagen' }) }); } else { await route.continue(); } })`
   - Navigate to Stammdaten tab, change the Schulname field, click Speichern
   - Assert red toast: `await expect(page.getByText(/Aenderungen konnten nicht gespeichert|Validierung/)).toBeVisible()`
   - Assert green toast NEVER fires: `await expect(page.getByText('Aenderungen gespeichert.')).not.toBeVisible({ timeout: 3000 })`

2. **Test `SILENT-4XX-02: Zeitraster PUT 4xx → red toast, no green toast`:**
   - `await loginAsAdmin(page)`
   - `await page.route('**/api/v1/schools/*/time-grid', route => route.fulfill({ status: 422, body: JSON.stringify({ statusCode: 422, message: 'Zeitraster ungueltig' }) }))`
   - Navigate to Zeitraster tab, trigger Speichern
   - Red toast visible, green toast NOT visible

3. **Test `SILENT-4XX-03: Schuljahr POST 4xx → red toast, no green toast`:**
   - `await loginAsAdmin(page)`
   - `await page.route('**/api/v1/schools/*/school-years', route => { if (route.request().method() === 'POST') { route.fulfill({ status: 422, body: JSON.stringify({ message: 'Datumsbereich ungueltig' }) }); } else { route.continue(); } })`
   - Open Neues Schuljahr dialog, fill minimally, submit
   - Red toast visible, green `Schuljahr angelegt` NOT visible

4. **Test `SILENT-4XX-04: Options A/B toggle PUT 4xx → red toast, no green toast`:**
   - `await loginAsAdmin(page)`
   - `await page.route('**/api/v1/schools/seed-school-bgbrg-musterstadt', async (route, request) => { if (request.method() === 'PUT' && (await request.postDataJSON())?.abWeekEnabled !== undefined) { await route.fulfill({ status: 422, body: JSON.stringify({ message: 'Option konnte nicht aktualisiert werden' }) }); } else { await route.continue(); } })`
   - Navigate to Optionen tab, click A/B toggle
   - Red toast visible, green `Option gespeichert.` NOT visible
</action>
