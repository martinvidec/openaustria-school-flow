## Decisions Made

- **Seed school ID hard-coded in SILENT-4XX-01 and SILENT-4XX-04:** The plan specifies the literal `seed-school-bgbrg-musterstadt` in route patterns. Hoisted to a `SEED_SCHOOL_ID` constant at top-of-file for single-point maintenance if the seed UUID ever drifts.
- **Payload narrowing for the Optionen mock:** `SILENT-4XX-04` matches only PUTs where `abWeekEnabled` is present in the JSON body. OptionsTab and SchoolDetailsTab share `PUT /schools/:id`, so narrowing prevents cross-test contamination.
- **`pressSequentially` over `fill` for the dirty-edit in SILENT-4XX-01:** The react-hook-form form in SchoolDetailsTab runs a `useEffect(reset(...))` on hydration. Using `fill` raced with that effect and left `isDirty=false`, disabling Speichern. Switching to `focus` + `press('End')` + `pressSequentially` triggers the same input events a real user would generate, and RHF tracks isDirty correctly.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Replaced `locator.fill()` with focus + pressSequentially in SILENT-4XX-01**

- **Found during:** Task 1 (first test run)
- **Issue:** Test 1 timed out (60s) clicking the Speichern button because the button was still `disabled`. Root cause: `nameInput.fill(...)` completed before SchoolDetailsTab's hydration `useEffect` fired `reset(...)`, which wiped my edit and re-marked the form clean. `isDirty` stayed false, Speichern stayed disabled.
- **Fix:** Wait for hydration signal (`await expect(nameInput).not.toHaveValue('')`) before editing. Then use `focus()` + `press('End')` + `pressSequentially(...)` to simulate real keyboard input — this fires the input/change events RHF needs to track dirty state correctly. Added `await expect(saveButton).toBeEnabled()` before the click to make any future re-break loud instead of timing out.
- **Files modified:** apps/web/e2e/silent-4xx.spec.ts (SILENT-4XX-01 only — tests 2/3/4 never had this problem because they dirty the form via `click` on toggles/buttons, not text input)
- **Verification:** SILENT-4XX-01 went from 60s-timeout to 1.7s green. All 9 Phase 10.2 desktop specs pass after the fix.
- **Committed in:** `6a69128` (Task 1 commit — fix included in initial spec commit)
