1. **Test `ZEIT-01: Happy-Path — add period, save, reload, assert persistence`:**
   - `await loginAsAdmin(page)`
   - `await page.goto('/admin/school/settings?tab=timegrid')`
   - Count existing periods (e.g., `const initialCount = await page.getByRole('button', { name: /Periode .* loeschen/ }).count()`)
   - Click `page.getByRole('button', { name: /Periode hinzufuegen/ })`

   - Click `page.getByRole('button', { name: 'Speichern' }).first()`
   - `await expect(page.getByText(/Aenderungen gespeichert/)).toBeVisible()`
   - `await page.reload()`
   - Assert the new period label is still visible in the editor: `await expect(page.getByDisplayValue('E2E-Period')).toBeVisible()` OR equivalent field query
   - Cleanup: remove the period via the delete button and save again so the spec is idempotent

2. **Test `ZEIT-02: Error-Path — invalid period (endTime before startTime) triggers red toast, no green toast`:**
   - `await loginAsAdmin(page)`
   - `await page.goto('/admin/school/settings?tab=timegrid')`
   - Use `page.route('**/api/v1/schools/*/time-grid', route => route.fulfill({ status: 422, contentType: 'application/json', body: JSON.stringify({ statusCode: 422, message: 'Endzeit muss nach Startzeit liegen' }) }))` to deterministically force a 4xx regardless of client-side validation
   - Click `page.getByRole('button', { name: /Periode hinzufuegen/ })` and submit any period
   - Click Save
   - `await expect(page.getByText('Aenderungen gespeichert')).not.toBeVisible({ timeout: 3000 })` — the critical silent-4xx guard
   - `await expect(page.getByText(/Endzeit muss nach|konnten nicht gespeichert/)).toBeVisible()` — red toast visible
</action>
