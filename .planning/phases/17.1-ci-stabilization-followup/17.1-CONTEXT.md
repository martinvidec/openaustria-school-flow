### TS build-blocker fix path (Plan 17.1-01)
- **D-05:** Root cause is **TanStack Query v5 discriminated-union type narrowing** when supplying a partial `UseQueryResult` mock. When tests pass `{isSuccess: true, data: ..., status: 'success'}`, the type system expects `status` to be the LITERAL `'success'`, not the union `Q['status']` (which expands to `'error' | 'pending' | 'success'`). Fix path: cast literally per discriminated union variant, OR use `as Partial<UseQueryResult>` cast at the mock-construction site.
- **D-06:** Acceptance criterion is `pnpm --filter @schoolflow/web exec tsc -p tsconfig.app.json --noEmit` → exit 0. NOT just "no errors in DashboardChecklist.test.tsx" — the whole web tsc must pass. If the fix surfaces OTHER pre-existing TS errors elsewhere, those become a deferred item to Phase 17.2 (do NOT scope-creep into this plan).
- **D-07:** Plan 17.1-01 produces a SINGLE atomic commit (`fix(dashboard): resolve UseQueryResult discriminated-union type narrowing in DashboardChecklist tests`). No bundled refactors. The DashboardChecklist mock factory MAY be lifted to `apps/web/src/test-utils/queryMocks.ts` IF doing so reduces duplication across 8 sites — but only if no other test files need touching.

### Worktree merge orchestration (Plan 17.1-02)
- **D-08:** Merge order is FIXED: `17-01` → `17-02` → `17-03` → `17-04` → `17-05`, in that sequence, against current main. Per memory `feedback_phase_branch_discipline.md` (parallel branches OK if geordnet gemerged) — these 5 worktree branches were authored in parallel; the merge MUST be sequential.
- **D-09:** Each branch merge is a SEPARATE PR (do NOT bundle into one mega-merge). Each PR title: `merge(17-NN): <plan-summary>` to preserve atomic-commit lineage. Each PR runs CI; if any RED → STOP, document, escalate (do NOT force-merge with `--admin` — that would re-violate the very gate Phase 17 was meant to deliver).
- **D-10:** After all 5 PRs merge to main, the smoke-PR `chore/ci-smoke-noop` is re-pushed (rebased onto new main HEAD) and CI re-triggered. Expected GREEN. If RED with NEW failures NOT in `deferred-items.md` cluster A–F list → document in `17-TRIAGE.md` `## Smoke-PR result` and continue to best-effort plans 17.1-03..17.1-05; if RED with cluster A–F failures → those plans MUST execute.
