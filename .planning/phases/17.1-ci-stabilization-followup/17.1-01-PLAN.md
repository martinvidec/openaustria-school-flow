<!-- Reference: https://tanstack.com/query/v5/docs/reference/useQuery -->

The `UseQueryResult<TData, TError>` type from @tanstack/react-query is a DISCRIMINATED UNION over the `status` field:

```typescript
type UseQueryResult<TData, TError> =
  | (QueryObserverPendingResult<TData, TError>  & { status: 'pending'; isPending: true; isSuccess: false; isError: false; data: undefined })
  | (QueryObserverSuccessResult<TData, TError>  & { status: 'success'; isPending: false; isSuccess: true; isError: false; data: TData })
  | (QueryObserverLoadingErrorResult<TData, TError> & { status: 'error'; isPending: false; isSuccess: false; isError: true; error: TError; data: undefined })
  | (QueryObserverRefetchErrorResult<TData, TError> & { status: 'error'; ... });
```

The current test file declares `type Q = UseQueryResult<DashboardStatusDto, Error>`. When passing `status: 'success' as Q['status']` to `Partial<Q>`, TypeScript expands `Q['status']` to the FULL UNION `'pending' | 'success' | 'error'`, which is NOT assignable to `Partial<{ status: 'success' }>` (the success variant requires the LITERAL).

Fix paths (pick the smallest-diff option):
1. Replace each `status: '<X>' as Q['status']` with `status: '<X>' as const` (literal-narrows in place; smallest possible diff — 8 edits, one per line).
2. Cast the entire partial object: change `makeQueryResult({...})` → `makeQueryResult({...} as Partial<Q>)` (8 edits, but each is just an outer cast — visual diff is slightly larger).

PREFER option 1 (`as const`) — it preserves type-safety for every other field while narrowing only the `status` literal. Option 2 is a fallback if `as const` triggers downstream errors on `data`/`error` shape.
</interfaces>
</context>
