  <behavior>
    HomeworkService:
    - create(): Creates homework record with classSubjectId, classBookEntryId (optional), title, dueDate. Returns HomeworkDto with joined subjectName/className.
    - create(): After successful create, calls NotificationService.create() for each student and parent in the class with type HOMEWORK_ASSIGNED. Excludes the creating teacher userId from recipients (self-notification prevention per RESEARCH anti-pattern).
    - findByClassSubject(): Returns homework[] for a classSubjectId, ordered by dueDate DESC.
    - findBySchool(): Returns homework[] for a schoolId with pagination.
    - update(): Updates title, description, dueDate. Returns updated record.
    - delete(): Soft delete or hard delete by id.

    ExamService:
    - checkCollision(classId, date, excludeId?): Queries exams where classId matches AND date is same calendar day. Returns { hasCollision: boolean, existingExam?: ExamDto }.
    - create(): Creates exam record. Before insert, calls checkCollision(). If collision exists, still creates (D-03: soft warning at API level, not hard block) but returns collision info in response.
    - create(): After successful create, calls NotificationService.create() for each student and parent in the class with type EXAM_SCHEDULED.
    - findByClass(): Returns exams[] for classId, ordered by date ASC.
    - update(): Updates fields, re-checks collision on date change.
    - delete(): Deletes exam by id.
  </behavior>
