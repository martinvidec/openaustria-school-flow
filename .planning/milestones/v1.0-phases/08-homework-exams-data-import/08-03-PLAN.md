4. Create ImportService (apps/api/src/modules/import/import.service.ts):
   - Injects PrismaService, @InjectQueue(IMPORT_QUEUE) importQueue: Queue.
   - uploadAndParse(schoolId, file: { filename, content }, userId): Detect file type (XML/DIF/CSV). Parse with appropriate parser. Store file content to a temp path using `path.join(os.tmpdir(), 'schoolflow-import-' + crypto.randomUUID())` and write via fs.writeFile. Store this filePath on the ImportJob record. Return parse summary (entity counts for Untis, headers+rows for CSV).
   - startDryRun(schoolId, importJobId): Create ImportJob record in QUEUED, add BullMQ job with ImportJobData including filePath from the ImportJob record and dryRun: true. Return job ID.
   - commitImport(importJobId): Update ImportJob status to PROCESSING. Add BullMQ job with ImportJobData including filePath and dryRun: false.
   - getJob(importJobId): Return ImportJob record.
   - getHistory(schoolId): Return ImportJob[] ordered by createdAt desc.
   - processImportRow(row, entityType, schoolId, conflictMode): Per-entity-type import logic:
     - TEACHERS: Find teacher by shortName or lastName+firstName. Skip/update/fail per conflictMode. Create Person+Teacher via nested Prisma create (Phase 2 pattern).
     - CLASSES: Find class by name. Skip/update/fail. Create SchoolClass.
     - ROOMS: Find room by name. Skip/update/fail. Create Room.
     - STUDENTS: Find student by lastName+firstName. Skip/update/fail. Create Person+Student.
   - Each processImportRow returns { status: 'imported'|'skipped'|'error', error?: string }.

5. Create ImportProcessor (apps/api/src/modules/import/processors/import.processor.ts):
   - @Processor(IMPORT_QUEUE) extends WorkerHost.
   - Injects ImportService and ImportEventsGateway.
   - async process(job: Job<ImportJobData>): Read file content from job.data.filePath using fs.readFile. Parse per file type. If dryRun: validate all rows without DB writes, store dryRunResult on ImportJob, set status to DRY_RUN. If not dryRun: process each row via importService.processImportRow, update progress via job.updateProgress() and gateway.emitProgress() per row. After all rows: update ImportJob with counts (imported, skipped, error). Set status to COMPLETED/PARTIAL/FAILED. Emit complete event. Clean up temp file via fs.unlink(job.data.filePath) in a finally block.
   - Set job timeout to 30 minutes (per Pitfall 6).
   - Process rows in chunks of 50 for batch efficiency.
