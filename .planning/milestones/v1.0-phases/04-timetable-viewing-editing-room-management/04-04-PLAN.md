    **3. Extend RoomService with booking and availability methods:**

    **bookRoom(schoolId, dto: CreateRoomBookingDto, userId: string) -> RoomBookingResponseDto:**
    1. Verify room belongs to schoolId
    2. Check for conflict: existing RoomBooking at same roomId + dayOfWeek + periodNumber + weekType. Also check TimetableLesson on the active run at same position. If conflict exists, throw ConflictException("Dieser Raum ist fuer die gewaehlte Zeit bereits belegt. Bitte waehlen Sie einen anderen Zeitpunkt.")
    3. Create RoomBooking with bookedBy = userId, isAdHoc = true, weekType = dto.weekType || 'BOTH'
    4. Return with room.name joined

    **cancelBooking(schoolId, bookingId: string, userId: string) -> void:**
    1. Find booking, verify room belongs to schoolId
    2. Verify bookedBy === userId OR user has admin/schulleitung role (pass role info)
    3. Delete the booking

    **getAvailability(schoolId, query: RoomAvailabilityQueryDto) -> RoomAvailabilitySlotDto[]:**
    1. Find all rooms for schoolId with optional filters (roomType, minCapacity, equipment contains)
    2. Find the school's TimeGrid periods (non-break only)
    3. For the specified dayOfWeek, build a grid: rooms x periods
    4. Query TimetableLesson (active run) for all rooms on that day
    5. Query RoomBooking for all rooms on that day
    6. Mark each slot as available or occupied with occupier details:
       - lesson: type 'lesson', label = subject abbreviation (join through ClassSubject -> Subject)
       - booking: type 'booking', label = purpose || 'Ad-hoc-Buchung', bookedBy = booking.bookedBy
    7. Return flat array of RoomAvailabilitySlotDto

    **getBookingsForRoom(roomId: string, dayOfWeek?: string) -> RoomBookingResponseDto[]:**
    Query RoomBooking where roomId, optional dayOfWeek filter, ordered by periodNumber.

    **PART B: WebSocket Timetable Events Gateway**

    **7. Create apps/api/src/modules/timetable/timetable-events.gateway.ts:**
    A NEW WebSocket gateway on the `/timetable` namespace (separate from the existing `/solver` namespace per Pitfall 2 in RESEARCH.md):

    ```typescript
    @WebSocketGateway({
      namespace: 'timetable',
      cors: { origin: '*' },
      transports: ['websocket', 'polling'],
    })
    export class TimetableEventsGateway implements OnGatewayConnection, OnGatewayDisconnect {
      @WebSocketServer()
      server!: Server;

      handleConnection(client: Socket) {
        const schoolId = client.handshake.query.schoolId as string;
        if (schoolId) client.join(`school:${schoolId}`);
      }

      handleDisconnect(_client: Socket) {
        // Socket.IO handles room cleanup automatically
      }

      emitTimetableChanged(schoolId: string, payload: {
        changeType: 'changed' | 'cancelled' | 'room-swap' | 'substitution';
        lessonId: string;
        changeCount: number;
      }): void {
        this.server.to(`school:${schoolId}`).emit(`timetable:${payload.changeType}`, payload);
      }

      emitRoomBookingChanged(schoolId: string, payload: {
        action: 'booked' | 'cancelled';
        roomId: string;
        roomName: string;
        dayOfWeek: string;
        periodNumber: number;
      }): void {
        this.server.to(`school:${schoolId}`).emit('timetable:room-swap', payload);
      }
    }
    ```
