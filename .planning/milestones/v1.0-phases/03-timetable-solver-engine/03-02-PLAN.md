7. Create `apps/solver/src/main/java/at/schoolflow/solver/constraints/TimetableConstraintProvider.java`:
   Implements ConstraintProvider with 4 hard constraints per D-02:

   a) `teacherConflict`: forEachUniquePair(Lesson.class, Joiners.equal(Lesson::getTimeslot), Joiners.equal(Lesson::getTeacherId)).penalize(HardSoftScore.ONE_HARD).asConstraint("Teacher conflict")

   b) `roomConflict`: forEachUniquePair(Lesson.class, Joiners.equal(Lesson::getTimeslot), Joiners.equal(Lesson::getRoom)).penalize(HardSoftScore.ONE_HARD).asConstraint("Room conflict")

   c) `teacherAvailability`: forEach(Lesson.class).join(TeacherAvailability.class, Joiners.equal(Lesson::getTeacherId, TeacherAvailability::getTeacherId), Joiners.equal(lesson -> lesson.getTimeslot().getDayOfWeek(), TeacherAvailability::getDayOfWeek), Joiners.equal(lesson -> lesson.getTimeslot().getPeriodNumber(), TeacherAvailability::getPeriodNumber)).penalize(HardSoftScore.ONE_HARD).asConstraint("Teacher availability")

   d) `studentGroupConflict`: forEachUniquePair(Lesson.class, Joiners.equal(Lesson::getTimeslot), Joiners.equal(Lesson::getClassId)).filter((l1, l2) -> l1.getGroupId() == null || l2.getGroupId() == null || l1.getGroupId().equals(l2.getGroupId())).penalize(HardSoftScore.ONE_HARD).asConstraint("Student group conflict")
   -- Two whole-class lessons at same time = clash. Whole-class + group = clash (students overlap). Same group at same time = clash. Different groups at same time = OK (disjoint students).

   Return Constraint[] from defineConstraints() containing all 4 constraints.

4. Create `apps/solver/src/main/java/at/schoolflow/solver/rest/SolverResource.java`:
   - @Path("/solve") @ApplicationScoped
   - Inject SolverManager<SchoolTimetable, String>
   - Inject SolutionManager<SchoolTimetable, HardSoftScore> for score analysis

   Endpoints:
   a) POST /solve -- accepts SolveRequest. Calls solverManager.solveBuilder()
      .withProblemId(runId)
      .withProblem(problem)
      .withBestSolutionConsumer(solution -> sendProgressCallback(callbackUrl, runId, solution))
      .withFinalBestSolutionConsumer(solution -> sendCompletionCallback(callbackUrl, runId, solution))
      .run()
      Override termination via SolverManager config with maxSolveSeconds from request.
      Return 202 Accepted with { runId, status: "SOLVING" }.

   b) GET /solve/{runId}/status -- return current solve status (SOLVING/COMPLETED/NOT_FOUND) by checking SolverManager.getSolverStatus(runId).

   c) DELETE /solve/{runId} -- terminate early via solverManager.terminateEarly(runId). Return 200 with { runId, status: "STOPPED" }.

   d) GET /health -- return 200 with { status: "UP" } for Docker healthcheck.

   Progress callback implementation:
   - Use java.net.http.HttpClient to POST SolveProgress JSON to callbackUrl/progress/{runId}
   - On final solution, POST SolveResult JSON to callbackUrl/complete/{runId}
   - Use SolutionManager.analyze() on final solution to build ViolationGroup list
   - Track score history in a ConcurrentHashMap<String, List<ScoreHistoryEntry>>
   - Calculate improvementRate by comparing last 3 scores: "improving" if improving, "plateauing" if < 1% change, "stagnant" if no change
