    **Step 1: Add pnpm peer dependency override (Pitfall 2)**
    In root `package.json`, add `peerDependencyRules` to the existing `pnpm` key:
    ```json
    "pnpm": {
      "onlyBuiltDependencies": [...existing...],
      "peerDependencyRules": {
        "allowedVersions": {
          "vite-plugin-pwa>vite": "8"
        }
      }
    }
    ```

    **Step 2: Install dependencies**
    Run in `apps/web`:
    ```bash
    pnpm add vite-plugin-pwa
    pnpm add -D workbox-precaching workbox-routing workbox-strategies workbox-expiration
    ```
    These Workbox packages are needed for the custom `sw.ts` imports in injectManifest mode (09-RESEARCH Open Question 3).

    **Step 3: Create placeholder PWA icons**
    Create simple placeholder PNG icons at `apps/web/public/icons/`:
    - `icon-192.png` (192x192) -- simple blue square with white "SF" text
    - `icon-512.png` (512x512) -- same design scaled up
    - `badge-72.png` (72x72) -- monochrome badge for notification badge
    Use a CLI tool (e.g., ImageMagick `convert` if available) or create minimal valid PNGs programmatically. If no image tool is available, create minimal 1x1 placeholder PNGs and note they should be replaced with proper branding.

    **Step 4: Configure VitePWA in vite.config.ts**
    Add VitePWA plugin to the existing Vite config using `injectManifest` strategy per 09-RESEARCH Pitfall 7 (generateSW does not support push event handlers):
    ```typescript
    import { VitePWA } from 'vite-plugin-pwa';

    // Add to plugins array:
    VitePWA({
      strategies: 'injectManifest',
      srcDir: 'src',
      filename: 'sw.ts',
      registerType: 'prompt',
      includeAssets: ['icons/*.png'],
      manifest: {
        name: 'SchoolFlow',
        short_name: 'SchoolFlow',
        description: 'Schulverwaltungsplattform',
        theme_color: '#3b82f6',
        background_color: '#ffffff',
        display: 'standalone',
        start_url: '/',
        lang: 'de',
        icons: [
          { src: 'icons/icon-192.png', sizes: '192x192', type: 'image/png' },
          { src: 'icons/icon-512.png', sizes: '512x512', type: 'image/png' },
          { src: 'icons/icon-512.png', sizes: '512x512', type: 'image/png', purpose: 'maskable' },
        ],
      },
      injectManifest: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
      },
      devOptions: {
        enabled: false,
      },
    })
    ```
    IMPORTANT: Add `navigateFallbackDenylist: [/^\/api\//]` in the injectManifest config to prevent Pitfall 3 (SW intercepting API requests and serving HTML).

    **Step 5: Create custom service worker (apps/web/src/sw.ts)**
    Follow 09-RESEARCH code example exactly. Include:
    - `/// <reference lib="webworker" />` at top
    - `precacheAndRoute(self.__WB_MANIFEST)` for app shell precache
    - `cleanupOutdatedCaches()` for old cache cleanup
    - `registerRoute` for timetable API with `NetworkFirst` strategy, 3-second timeout, 24-hour expiry, cacheName 'timetable-api', URL pattern matching `/api/v1/schools/[^/]+/timetable/view`
    - `registerRoute` for static assets (image, font destinations) with `CacheFirst`, 30-day expiry
    - `self.addEventListener('push', ...)` handler for push notifications (MOBILE-02 will use this in Plan 04). Show notification with title, body, icon, badge, tag from push event data. Fallback: `{ title: 'SchoolFlow', body: 'Neue Benachrichtigung' }`.
    - `self.addEventListener('notificationclick', ...)` handler to focus/open window at notification URL
    - `self.addEventListener('message', ...)` handler for `SKIP_WAITING` message (service worker update)

    **Step 6: Create useOnlineStatus hook**
    At `apps/web/src/hooks/useOnlineStatus.ts`:
    Use `useSyncExternalStore` with `window.addEventListener('online'/'offline')` subscription. `getSnapshot` returns `navigator.onLine`. Server snapshot returns `true`.

    **Step 7: Create usePwaInstall hook**
    At `apps/web/src/hooks/usePwaInstall.ts`:
    - Listen for `beforeinstallprompt` event, store deferred prompt in ref
    - Track `isInstallable` (prompt captured), `isInstalled` (display-mode: standalone check or appinstalled event)
    - Expose `install()` function that calls `prompt()` on the deferred event
    - Track session dismissal via `sessionStorage.setItem('pwa-install-dismissed', 'true')`

    **Step 8: Create useServiceWorker hook**
    At `apps/web/src/hooks/useServiceWorker.ts`:
    - Import `registerSW` from `virtual:pwa-register` (vite-plugin-pwa virtual module)
    - Register SW with `onNeedRefresh` callback (sets state indicating update available)
    - Expose `updateAvailable` state and `updateServiceWorker()` function
    - `updateServiceWorker()` posts `SKIP_WAITING` message then reloads
