    **GroupMembershipRuleService (D-07):**
    - Injectable service for auto-deriving group memberships from configurable rules
    - `async applyRules(classId: string, rules: GroupAutoAssignRule[])`:
      - Loads all students in the class
      - For each rule (type: 'RELIGION' | 'LEISTUNG' | 'LANGUAGE'):
        - Creates groups if they don't exist (e.g., "3B-Ethik", "3B-Katholisch", "3B-Deutsch-Standard", "3B-Deutsch-AHS")
        - Assigns students to groups based on rule logic
        - Only creates GroupMembership with isAutoAssigned=true
        - Preserves existing manual assignments (isAutoAssigned=false)
      - Returns summary: { groupsCreated: number, membershipsCreated: number }
    - `async clearAutoAssignments(classId: string)`:
      - Deletes all GroupMembership where isAutoAssigned=true for groups in this class
      - Does NOT touch manual assignments
    - GroupAutoAssignRule interface:
      ```typescript
      interface GroupAutoAssignRule {
        groupType: 'RELIGION' | 'LEISTUNG' | 'LANGUAGE';
        groupName: string;
        level?: string;          // For LEISTUNG
        studentFilter?: {        // Which students to include
          studentIds?: string[];  // Explicit list
        };
      }
      ```
    - For Phase 2, rules are passed as input (admin calls API with rules). Automatic inference from student attributes (e.g., religion field) is out of scope for now -- the admin provides the grouping.
