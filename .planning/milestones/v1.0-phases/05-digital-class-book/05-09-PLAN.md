**0. Update `apps/web/src/lib/api.ts` for FormData support (Pitfall 6):**

The current apiFetch sets Content-Type to application/json when body is present. For FormData uploads, the browser must set the Content-Type header with the multipart boundary. Add a FormData check:

```typescript
if (options?.body && !(options.body instanceof FormData) && !headers.has('Content-Type')) {
  headers.set('Content-Type', 'application/json');
}
```

This ensures FormData bodies get the correct browser-generated multipart/form-data Content-Type.
