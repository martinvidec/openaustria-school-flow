**2. Create `apps/api/src/modules/classbook/statistics.service.ts`:**

Inject PrismaService. Methods:

- `getClassStatistics(schoolId: string, query: AbsenceStatisticsQueryDto)`: Aggregate per-student absence statistics for a class. Steps:
  1. Resolve date range: if startDate not provided, default to current Austrian semester start (September 1 if today >= Sep 1, else February 1). If endDate not provided, default to today.
  2. Fetch all students in the class: `prisma.student.findMany({ where: { classId: query.classId }, include: { person: { select: { firstName: true, lastName: true } } } })`
  3. Fetch all ClassBookEntries for this class's classSubjects within date range: join through ClassSubject where classId matches
  4. Fetch all AttendanceRecords for those entries: `prisma.attendanceRecord.findMany({ where: { classBookEntryId: { in: entryIds }, studentId: { in: studentIds } } })`
  5. For each student, aggregate:
     - `totalLessons`: count of ClassBookEntries where the student had an AttendanceRecord (or count of entries for the class)
     - `presentCount`: records where status = PRESENT
     - `absentUnexcusedCount`: records where status = ABSENT (no linked excuse)
     - `absentExcusedCount`: records where status = EXCUSED
     - `lateCount`: records where status = LATE
     - `lateOver15MinCount`: records where status = LATE AND lateMinutes > 15 (D-04, Pitfall 7)
     - `absenceRate`: `((absentUnexcusedCount + absentExcusedCount + lateOver15MinCount) / totalLessons * 100)` -- per Austrian Schulunterrichtsgesetz, >15min late counts as absent
  6. Return AbsenceStatisticsDto[] sorted by student lastName, firstName

- `getStudentStatistics(schoolId: string, query: StudentAbsenceQueryDto)`: Single student statistics for parent/student view. Same aggregation but for one student. Returns a single AbsenceStatisticsDto.

- `getSemesterDateRange()`: Helper returning `{ start: Date, end: Date }` for the current Austrian school semester. Logic: if current month >= 9 (September) or current month == 1, semester is Sep 1 to Jan 31. If current month >= 2 and current month <= 6, semester is Feb 1 to Jun 30. July/August defaults to the previous full year (Feb 1 to Jun 30).

import { IsString, IsEnum, IsOptional, IsDateString, MaxLength } from 'class-validator';
import { ApiProperty, ApiPropertyOptional } from '@nestjs/swagger';

export enum ExcuseReasonEnum {
  KRANK = 'KRANK',
  ARZTTERMIN = 'ARZTTERMIN',
  FAMILIAER = 'FAMILIAER',
  SONSTIG = 'SONSTIG',
}

export enum ExcuseStatusEnum {
  PENDING = 'PENDING',
  ACCEPTED = 'ACCEPTED',
  REJECTED = 'REJECTED',
}

export class CreateExcuseDto {
  @ApiProperty({ description: 'Student ID (child)' })
  @IsString()
  studentId!: string;

  @ApiProperty({ description: 'Start date (ISO)' })
  @IsDateString()
  startDate!: string;

  @ApiProperty({ description: 'End date (ISO), must be >= startDate' })
  @IsDateString()
  endDate!: string;

  @ApiProperty({ enum: ExcuseReasonEnum, description: 'Reason category (D-11)' })

  @IsOptional()

  @IsOptional()

export class ExcuseListQueryDto {

**3. Create `apps/api/src/modules/classbook/excuse.service.ts`:**

Inject PrismaService. Methods:

- `createExcuse(schoolId: string, parentId: string, dto: CreateExcuseDto)`: Create AbsenceExcuse with PENDING status. Validations:
  - endDate >= startDate (throw BadRequestException if not)
  - startDate not more than 30 days in the past (throw BadRequestException)
  - Verify parent has the studentId as a child: `prisma.parentStudent.findFirst({ where: { parentId, studentId: dto.studentId } })`. If not found, throw ForbiddenException('Can only submit excuses for own children') (D-11, BOOK-06).
  Return the created excuse with parentName and studentName joined.

- `reviewExcuse(excuseId: string, reviewerKeycloakId: string, dto: ReviewExcuseDto)`: Review an excuse (Klassenvorstand only). Validations:
  - Excuse must be in PENDING status (throw ConflictException if not)
  - If dto.status is REJECTED, reviewNote is required (throw BadRequestException)
  - Verify reviewer is Klassenvorstand for the student's class: look up the excuse's student -> class -> SchoolClass.klassenvorstandId -> Teacher.personId -> Person.keycloakUserId. Compare to reviewerKeycloakId. Also allow admin/schulleitung roles.

  If ACCEPTED (D-12):
    1. Update excuse status to ACCEPTED, set reviewedBy, reviewNote, reviewedAt
    2. Find all ClassBookEntries in the date range (startDate to endDate) where the student's class has lessons: `prisma.classBookEntry.findMany({ where: { date: { gte: excuse.startDate, lte: excuse.endDate }, schoolId: excuse.schoolId } })`
    3. Filter entries to only those relevant to the student (matching class through classSubjectId -> ClassSubject -> classId -> student.classId)
    4. For each matching entry, update AttendanceRecord where studentId matches AND status is ABSENT: set status = EXCUSED, excuseId = excuse.id
    5. Run all updates in a Prisma transaction

  If REJECTED:
    1. Update excuse status to REJECTED, set reviewedBy, reviewNote, reviewedAt

- `getExcusesForParent(parentId: string)`: List excuses submitted by this parent. Join studentName. Order by createdAt DESC.

- `getPendingExcusesForKlassenvorstand(teacherKeycloakId: string, schoolId: string)`: Find the teacher's Person ID via keycloakUserId, then find their Teacher record, then find SchoolClasses where klassenvorstandId = teacher.id. For those classes, find students, and return pending excuses for those students. Order by createdAt ASC (oldest first).

- `getExcuseById(excuseId: string)`: Return full excuse with attachments, studentName, parentName.

- `saveAttachment(excuseId: string, filename: string, mimeType: string, sizeBytes: number, buffer: Buffer)`: Save file to disk at `uploads/{schoolId}/excuses/{excuseId}/{filename}`. Create directory if not exists (use `node:fs/promises` mkdir with recursive). Create ExcuseAttachment record. Validate mime type against allowed list: `['application/pdf', 'image/jpeg', 'image/png']`. Validate file magic bytes: PDF starts with `%PDF` (0x25504446), JPEG starts with 0xFFD8FF, PNG starts with 0x89504E47. Throw BadRequestException for invalid types.

- `getAttachmentPath(attachmentId: string)`: Return the storagePath for serving the file.
